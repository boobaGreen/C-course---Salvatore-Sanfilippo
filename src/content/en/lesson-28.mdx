## Anatomy of a Variadic Function

To declare a variadic function, you need at least one "fixed" argument followed by the ellipsis `...`.

```c
#include <stdarg.h>

void log_error(const char *fmt, ...) {
    va_list ap;
    va_start(ap, fmt); // Starts after 'fmt'
    
    // ... extracting arguments ...
    
    va_end(ap); // Mandatory cleanup
}
```

<Callout type="warning" title="The Importance of va_end">
Never forget `va_end(ap)`. While it does nothing on many architectures, on some platforms it's critical to avoid stack corruption or memory leaks.
</Callout>

---

## The Delegation Pattern: `vprintf`

There is a limit to variadic functions: you cannot pass the three dots (`...`) from one function to another like a normal variable. If you want to write a wrapper for `printf`, you must use its "sister" version that accepts a `va_list`: the `vprintf` function.

## Practical Example: Centered Text

Salvatore shows how to use these functions to manipulate output creatively, for example, by centering text in the terminal.

<CodeEditor 
  initialCode={`void print_centered(int width, const char *fmt, ...) {
    char buffer[1024];
    va_list ap;
    
    va_start(ap, fmt);
    vsnprintf(buffer, sizeof(buffer), fmt, ap);
    va_end(ap);
    
    int len = strlen(buffer);
    int padding = (width - len) / 2;
    
    for (int i = 0; i < padding; i++) putchar(' ');
    printf("%s\\n", buffer);
}`}
  language="c"
/>

<Callout type="warning" title="Type Safety">
Warning: `va_arg` has no idea what is actually on the stack. If you tell it to extract an `int` but the user passed a string, the program won't give a compilation error, but it will **Segmentation Fault** or corrupt memory at runtime!
</Callout>

## Extracting Arguments with `va_arg`

In C, the function doesn't automatically know "how many" arguments were passed. We must tell it through a format (like in printf) or a terminator.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdarg.h>

void print_numbers(int count, ...) {
    va_list ap;
    va_start(ap, count);
    
    for (int i = 0; i < count; i++) {
        int num = va_arg(ap, int); // Specify the TYPE!
        printf("Arg %d: %d\\n", i, num);
    }
    
    va_end(ap);
}

int main() {
    print_numbers(3, 10, 20, 30);
    return 0;
}`}
  language="c"
/>

<Callout type="hacker" title="Zero Type Safety">
Warning: `va_arg` doesn't check if the type is correct! If you pass a `double` but ask for an `int`, you'll get garbage values or a crash. It is the programmer's responsibility to ensure consistency.
</Callout>
