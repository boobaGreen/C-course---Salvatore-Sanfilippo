## The Heart of the Interpreter: `eval()`

The `eval` function is responsible for deciding what to do with a ToyFort object. If it's a value (integer, string), it pushes it onto the stack. If it's a symbol, it looks up the function in the dictionary and executes it.

## Implementing Built-in Functions

Every function in ToyFort must have a C counterpart. These functions interact directly with the `TFContext` (the stack).

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <string.h>

// Example implementation of the '+' (plus) function
void tf_plus(TFContext *ctx) {
    // 1. Pop the last two elements
    TFObject *b = tf_stack_pop(ctx);
    TFObject *a = tf_stack_pop(ctx);

    if (a->type == TF_INT && b->type == TF_INT) {
        // 2. Calculate the result
        long long res = a->i + b->i;
        // 3. Create a new object and PUSH it onto the stack
        tf_stack_push(ctx, create_int_object(res));
    } else {
        fprintf(stderr, "Runtime Error: + expects two integers\\n");
    }

    // 4. Memory management (Reference Counting)
    tf_object_release(a);
    tf_object_release(b);
}
`} 
  language="c" 
/>

<Callout type="warning" title="Memory Safety">
Remember: every time you `pop` from the stack, you are taking ownership of that pointer. It is critical to call `release` (or `free`) at the end of the function to avoid memory leaks!
</Callout>

## Parsing Lists and Programs

In ToyFort, a program is itself a **list** of objects. This allows for the implementation of conditionals and loops by passing entire lists as parameters to other functions.
