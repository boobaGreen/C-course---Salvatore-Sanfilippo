## The Professional Struct

Forget manual casts and skipping 4 or 8 bytes. C offers us an elegant tool: **Flexible Array Members**.

```c
struct pls {
    uint32_t len;        // 4 bytes: Length
    uint32_t ref_count;  // 4 bytes: Reference counter
    uint32_t magic;      // 4 bytes: Magic number for safety
    char str[];          // Flexible Array: points to offset 12!
};
```

### "Compact" Memory Layout

Thanks to `char str[]`, the header and text live in a **single contiguous block**. This is vital for CPU cache and allocation simplicity.

## Implementation: The `pls` Library

Here is the heart of the dynamic management discussed by Salvatore.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
#include <string.h>

#define PLS_MAGIC 0xDEADBEEF

struct pls {
    uint32_t len;
    uint32_t ref_count;
    uint32_t magic;
    char str[];
};

// Create a string with ref_count = 1
struct pls* ps_create(const char *init) {
    uint16_t lens = strlen(init);
    struct pls *p = malloc(sizeof(struct pls) + lens + 1);
    p->len = lens;
    p->ref_count = 1;
    p->magic = PLS_MAGIC;
    strcpy(p->str, init);
    return p;
}

// Increment confidence in the object
void ps_retain(struct pls *p) {
    if (p->magic != PLS_MAGIC) abort(); // Defense!
    p->ref_count++;
}

// Decrement and free if necessary
void ps_release(struct pls *p) {
    if (p->magic != PLS_MAGIC) abort();
    p->ref_count--;
    if (p->ref_count == 0) {
        p->magic = 0; // Invalidate before freeing
        free(p);
        printf("Object physically destroyed.\\n");
    }
}

int main() {
    struct pls *s1 = ps_create("Hacker");
    
    // Simulate sharing
    struct pls *s2 = s1; 
    ps_retain(s2); // Now ref_count is 2
    
    printf("String: %s (Refs: %d)\\n", s1->str, s1->ref_count);
    
    ps_release(s1); // ref_count returns to 1
    ps_release(s2); // ref_count goes to 0 -> FREE!
    
    return 0;
}`} 
  language="c" 
/>

<Callout type="hacker" title="Magic Numbers">
Why does Salvatore use `0xDEADBEEF`? It's easy to recognize in a hexadecimal dump (hexdump). If you see `00000000` where the magic should be, you know you're using an object that's already been freed!
</Callout>
