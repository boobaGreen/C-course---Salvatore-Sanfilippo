## Why Buffering? The Physics of Speed

Going into the Kernel (Ring 0) is like going through customs: you have to stop, show your papers, and wait. If you do this for every single character, your program slows down 100 times.

## mmap: The Ultimate Technique

With `mmap`, we ask the Kernel to link a file directly to a memory address in our process. We no longer need `read()` or `write()`: just read from the pointer!

<Callout type="pro" title="Zero-Copy Architecture">
  Normally, `read()` copies data from disk to Kernel, then from Kernel to your
  RAM buffer. `mmap` eliminates this second copy. Your pointer points directly
  to the memory pages managed by the Kernel (**Page Cache**). It's the secret
  speed of Redis and modern database engines.
</Callout>

### The Magic of the Page Fault

When you map a 1GB file, the Kernel doesn't load it all at once. It only creates a "map." When your code touches an address in that gigabyte, the hardware generates a **Page Fault**. The Kernel wakes up, loads only that page (4KB) from disk, and gives control back to your program. It's a perfect illusion of infinite and immediate memory.

## Implementation: Direct File Access

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
int fd = open("test.txt", O_RDONLY);
if (fd == -1) {
printf("Create a test.txt file to try!\\n");
return 1;
}

    struct stat st;
    fstat(fd, &st); // File size from Kernel

    // MAP_PRIVATE: changes stay in RAM (Copy-on-Write)
    // PROT_READ: read-only
    char *data = mmap(NULL, st.st_size, PROT_READ, MAP_PRIVATE, fd, 0);

    if (data == MAP_FAILED) {
        perror("mmap failed");
        return 1;
    }

    // The file is now an array!
    for (int i = 0; i < st.st_size; i++) {
        putchar(data[i]);
    }

    munmap(data, st.st_size); // Disconnect the memory
    close(fd);
    return 0;

}`}
