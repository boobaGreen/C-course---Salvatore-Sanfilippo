## Why Buffering?

Imagine you want to write a million characters.
1. If you call `write()` for every character, you will make 1 million syscalls (extremely slow).
2. If you use `fprintf()`, the library fills an 8KB buffer and calls `write()` only when it is full (much faster).

---

## mmap: The File is Your RAM

With `mmap`, we ask the Kernel to link a file directly to a memory address in our process. We no longer need `read()` or `write()`: just read from the pointer!

### Example: Reading a File with mmap

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd = open("test.txt", O_RDONLY);
    struct stat st;
    fstat(fd, &st); // Finding the file size

    // Map the file into read-only memory
    char *data = mmap(NULL, st.st_size, PROT_READ, MAP_PRIVATE, fd, 0);

    if (data == MAP_FAILED) {
        perror("mmap failed");
        return 1;
    }

    // Now we can read the file as if it were a string!
    printf("File content: %s\\n", data);

    munmap(data, st.st_size);
    close(fd);
    return 0;
}`} 
  language="c" 
/>
