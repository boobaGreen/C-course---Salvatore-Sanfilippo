## Why Reference Counting? Shared Ownership

The problem with C is that `free()` destroys everything. If two parts of a program use the same object, who should free it?

<Callout type="pro" title="The Redis Example">
  Salvatore uses Reference Counting in **Redis**. In a database, the same value
  can be referenced by thousands of keys. Copying the data would be RAM suicide.
  Refcount allows for **1 physical copy** and **N logical owners**.
</Callout>

## Under the Hood: Physical Overhead

An object with a refcount requires extra space. If your data is an `int` (4 bytes), adding a `refcount` (8 bytes) triples the memory usage for that object!
