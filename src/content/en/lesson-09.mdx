Salvatore recommends writing `int *p` instead of `int* p`. Why? If you declare `int *p, z;`, only `p` is a pointer, while `z` is a normal integer. Placing the asterisk near the variable name makes this C grammatical distinction clear.
</Callout>

## Why do Pointers have a Type?

A memory address is always the same size (64 bits on modern architectures), but the compiler needs to know **how many bytes to read** starting from that address.

- `int *p`: Read 4 bytes and interpret them as an integer.
- `char *c`: Read just 1 byte.

In C, address **0** is special and is indicated with the `NULL` keyword. It means the pointer doesn't point to anything valid.

<Callout type="hacker" title="Pointers vs Arrays: The Equivalence">
Did you know that `*p` is identical to writing `p[0]`? There's a "bizarre analogy" (as Salvatore calls it) in C between pointers and arrays: a pointer can be treated as the base address of a single-element array.
</Callout>

<Callout type="hacker" title="The NULL Pointer">
Salvatore always recommends initializing pointers to `NULL` (address 0). Accessing a pointer that points to nothing causes a **Segmentation Fault**, but an immediate error is better than a silent bug that writes into random memory areas!
</Callout>

## Code Example: Pass by Reference

Functions in C normally receive **copies** of values. To allow a function to modify the original variable, we must pass its "home address".

<CodeEditor 
  initialCode={`#include <stdio.h>

void increment(int *p) {
    *p = *p + 1; // Dereference p and add 1 to the original value
}

int main() {
    int x = 5;
    printf("Before: %d\\n", x);
    
    increment(&x); // We pass the address of x
    
    printf("After: %d\\n", x); // x is now 6!
    return 0;
}`}
  language="c"
/>
