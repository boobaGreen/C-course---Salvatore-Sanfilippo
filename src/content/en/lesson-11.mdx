## Beyond Data Types

In previous lessons, we've seen that pointers not only know *where* data is located, but also *what type* it is. This is crucial because it tells the compiler **how many bytes to read**.

But what happens if we "trick" the compiler by forcing a pointer to read memory in a different way? In C, we achieve this using **Casting**.

<MemoryCaster />

### What is Casting?

Casting is an operation that explicitly tells the compiler to treat a variable as if it were a different type.

```c
short s = 514;          // Occupies 2 bytes
char *p = (char *)&s;   // Force a char* pointer to look at the start of the short
```

If `s` has the value `514` (which is `0x0202` in Hex), its physical memory contains two separate bytes: `0x02` and `0x02`.
When we read `*p`, we are reading only **the first byte** (`0x02`, which is `2` in decimal), because `p` is a pointer to a `char` (1 byte).

<Callout type="hacker">
**Type Punning**
This technique of reading a type's memory through a pointer of another type is called "Type Punning". It is powerful for systems programming, but it can cause undefined behavior if strict aliasing and alignment rules are not respected.
</Callout>

### Endianness: The Byte Wars
**Endianness** defines the order in which the bytes composing a multi-byte number (like a 2-byte `short` or a 4-byte `int`) are stored in physical memory.

* **Little Endian** (Intel/AMD x86 architectures): The *Least Significant Byte* (LSB) is stored at the lowest memory address.
* **Big Endian** (TCP/IP Network protocols, older Motorola CPUs): The *Most Significant Byte* (MSB) is stored at the lowest address.

Example using the number `0x11223344`:
* In **Little Endian** memory: `44 33 22 11` (base address holds `44`).
* In **Big Endian** memory: `11 22 33 44` (base address holds `11`).

## C Strings vs Length-Prefixed Strings

Until now, we've used classic C strings (known as *Null-terminated strings*). They end with the `0x00` byte.

However, these strings have geometric problems:
1. To find the length, you must scan the entire string (Complexity `O(N)`).
2. They cannot contain `0x00` inside them, because it would be interpreted as the end!

A historical alternative (made famous by the Pascal language) is the **Length-Prefixed String**.
In this format, the **first byte** stores the total length of the string.

```c
// Pascal String. The first byte (11) dictates the length.
char *pascal_str = "\x0BHello World"; // \x0B is 11 in hex.
```

This makes getting the length an instantaneous operation (`O(1)`), simply by reading `pascal_str[0]`. The downside of using a single byte? A maximum length of 255 characters!

## Editor Code
This example shows what happens when manipulating characters and pointers at the byte level.

<CodeEditor 
  initialCode={`#include <stdio.h>

int main() {
    // Pascal-style string: first char = 5 (length), then "Hello"
    char str[] = {5, 'H', 'e', 'l', 'l', 'o'};
    
    // Read the length:
    int len = str[0];
    
    printf("The string is %d characters long.\\n", len);
    
    // Print the remaining characters
    for(int i = 1; i <= len; i++) {
        putchar(str[i]);
    }
    printf("\\n");
    
    // EXPERIMENT: Cast to short
    short *s = (short *)str;
    printf("Value of the first short in hex memory: 0x%04X\\n", s[0]);
    // Hint: In a Little Endian system (x86), it will assemble 'H' (0x48) as the MSB
    // and 5 (0x05) as the LSB -> 0x4805.
    
    return 0;
}`}
