## Registers vs RAM: The Two Pillars

All microprocessors (from the legendary **6502** in the Commodore 64 to the **M3** in a modern MacBook) have two ways of processing data:

1. **CPU Registers**: Tiny variables (a few bytes) located *inside* the processor core. They are instantaneous.
2. **RAM (Memory)**: A large external warehouse that is much slower.

<Callout type="info" title="Speed and Efficiency">
CPU instructions can operate directly on registers. To use data in RAM, the processor must often "load" it into a register, perform the calculation, and then "save" it back to RAM.
</Callout>

## 1. The World of Registers (The 6502 Model)

The 6502 is a 40-year-old processor, perfect for learning because it is incredibly simple. It has only three main registers: **A** (Accumulator), **X**, and **Y**.

### Simulator: Writing to Video Memory
In this architecture, the screen is **Memory Mapped**. This means some RAM cells are physically connected to the screen. If we write the value `5` to address `$0200`, that pixel will turn **Green** (use the **Legend** below to experiment with colors).

<MicroRegisters />

Here is the assembly code Salvatore wrote to fill the screen:
```assembly
LDA #$05    ; Load color 5 (Green) into register A
TAX         ; Transfer A to register X (we'll use it as a counter)
LDY #$00    ; Clear register Y (position index)

LOOP:
  STA $0200, Y  ; Store the color at address $0200 + Y
  INY           ; Increment Y (next pixel)
  DEX           ; Decrement X (how many left?)
  BNE LOOP      ; If X is not zero, continue the loop
```

<Callout type="pro" title="Ephemeral Variables">
Notice how registers are continuously overwritten. If one function calls another and that one uses register **A**, the original value is lost forever unless we save it somewhere first.
</Callout>

### Ghosts (Garbage) in the Stack
When a function finishes (`return`), the Stack Pointer moves back. The "drawer" is not physically emptied for speed reasons. The data stays there until another function writes over it.
If you declare `int x;` without giving it a value, you'll read the **"ghosts"** of previous functions!

## 3. The Calling Convention (Details)

Salvatore shows how the compiler transforms `sum(10, 20)` into assembly.
1. **Push 20**: Puts the second argument onto the Stack.
2. **Push 10**: Puts the first argument onto the Stack.
3. **Call sum**: Jumps to the function while saving the return address.
4. **Prologue**: The function saves the `Base Pointer` (EBP) to know where its arguments are located.
