## Loops: for vs while
As Salvatore explains, a `for` loop is often just a more compact way of writing a `while`.

```c
// WHILE Version
int i = 0;
while (i < 5) {
    // code
    i++;
}

// FOR Version (cleaner)
for (int i = 0; i < 5; i++) {
    // code
}
```

We use `for` when we already know how many times to iterate, and `while` when the condition depends on something that will happen in the code.

<Callout type="hacker" title="For: Total Flexibility">
In C, the three parts of the `for` loop (initialization, condition, increment) are optional!
- `for(;;)` is an idiomatic way to write an **infinite loop**.
- You can omit the increment if you handle it inside the loop body.
</Callout>

### Practical Example: Array Search
Salvatore shows how to use `break` to stop a loop as soon as we find what we're looking for.

```c
int a[] = {1, 5, 8, 4, 0}; // 0 is the terminator
int i;
for (i = 0; a[i] != 0; i++) {
    if (a[i] == 8) break; // Found it! Exit immediately.
}

if (a[i] == 0) printf("Not found\n");
else printf("Found at position %d\n", i);
```

<Callout type="warning" title="Variable Scope of i">
If you declare `for (int i = 0; ...)` inside the loop, the variable `i` disappears as soon as the loop ends. If you need to know where the loop stopped (like in the example above), you must declare `i` **outside** the `for`.
</Callout>

## Recursion and the Stack

When a function calls itself, the computer doesn't start from scratch. It "freezes" the current function and opens a new one on top of it (a new "frame").

<Callout type="hacker" title="Tail Call Optimization (TCO)">
If the last thing a function does is call itself, a good compiler can optimize it by turning it into a jump (`goto`), saving memory. If you don't trust the compiler, you can do it manually:
```c
loop:
    // ... do something ...
    start = start + 1;
    goto loop; // Manual recursion!
```
</Callout>

<RecursionVisualizer />

<Callout type="warning" title="Stack Overflow">
If recursion has no "exit condition" (base case), it will keep creating new frames until it runs out of memory dedicated to the stack. This causes the famous 'Stack Overflow' crash.
</Callout>

## Choosing Elegantly: The Switch

If you have a variable that can be 1, 2, 3... instead of writing ten `if`s, we use `switch`. It's more readable and often faster!

<SwitchBoard />

<Callout type="hacker" title="Jump Tables">
Under the hood, if the cases are close together (e.g., 1, 2, 3), the compiler doesn't do many `if` checks. It creates a **Jump Table**: a table of addresses where the processor "jumps" instantly to the correct case. It's incredibly fast!
</Callout>

<Callout type="info" title="The Importance of Break">
In a `switch`, if you forget the `break;`, the code will continue to execute subsequent cases as well! This behavior is called 'fall-through'.
</Callout>

## Real Code: Redis Glob Matching
Salvatore shows how he uses these concepts in real production code. His string matching function (e.g., handles `*`, `?`, `[a-z]`) uses both `switch` and recursion to handle various pattern characters.

```c
// Simplified version of Salvatore's glob matching
int glob_match(const char *pattern, const char *string) {
    switch (pattern[0]) {
        case '*': // Recursion to handle the asterisk
            // ... complex logic ...
            break;
        case '?': // Skip one character
            // ...
            break;
        default: // Literal match
            if (pattern[0] != string[0]) return 0;
    }
}
```

  - **for = while**: The `for` loop is just a more compact way to write a `while` with initialization and increment.
  - **Stack vs Recursion**: Every recursive call occupies space on the Stack (frame). Be careful not to exhaust it!
