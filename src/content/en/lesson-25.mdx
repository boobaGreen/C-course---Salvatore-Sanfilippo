## Visualizing Objects: The Power of Recursion

Before we can read, we must know how to write. A ToyFort list can contain other lists. To print them correctly, our `print_object` function must be recursive.

<CodeEditor 
  initialCode={`void print_object(TFObject *o) {
    switch(o->type) {
        case TF_INT:
            printf("%lld", o->i);
            break;
        case TF_LIST:
            printf("[ ");
            for (size_t i = 0; i < o->l.len; i++) {
                print_object(o->l.ele[i]); // RECURSIVE call
                if (i < o->l.len - 1) printf(" ");
            }
            printf(" ]");
            break;
        case TF_SYM:
            printf("%s", o->s.ptr);
            break;
    }
}`}
  language="c"
/>

<Callout type="warning" title="The Newline Secret">
Salvatore notes that without a final `\\n`, the shell (like Z-shell or `zsh`) might show a `%` character or an inverted symbol at the end of the output. This is why we add a `printf("\\n")` only in the main caller, not inside the recursive function!
</Callout>

<Callout type="info" title="Why Recursion?">
It's not just an academic exercise! Recursion is the most natural way to navigate tree-like structures like programs. Without it, we would have to manually manage a complex stack.
</Callout>

## Implementing the Symbol Parser

A symbol in ToyFort is any sequence of characters that isn't a number or a string. We need to define a set of acceptable characters (alphanumeric + mathematical symbols).

### The `is_symbol_char` function

Salvatore uses an elegant trick with `strchr` to check if a character belongs to a set of allowed symbols:

```c
int is_symbol_char(int c) {
    if (isalpha(c)) return 1;
    if (strchr("+-*/%=<>!?", c)) return 1;
    return 0;
}
```
