## Beyond the Limits of the Stack

Until now, every variable we created lived on the **Stack**. The stack is fast and safe, but it has two major flaws:

1. You must know the size of the variables **at compile time**.
2. When a function finishes, **its entire stack is destroyed**.

If we want to create data that "survives" across functions, or data whose size is only known while the program is running (e.g., a text file loaded by the user), we must use the **Heap**.

To access the Heap, we use functions from the standard `<stdlib.h>` library:

<Callout type="info" title="Malloc: A Contract with the OS">
  When you call `malloc`, you are asking the Operating System for a piece of
  RAM. If the OS accepts, it gives you a memory address (`void *`). From that
  moment on, **you are the master** of those bytes until you call `free`. If you
  forget to free them, that memory remains "dead" to the rest of the computer
  until your program closes.
</Callout>

```c
#include <stdlib.h>

// I want space for 100 integers (100 * 4 bytes = 400 bytes)
int *numbers = malloc(100 * sizeof(int));

if (numbers == NULL) {
    // RAM is full! The OS said no.
    return 1;
}

// ... use the numbers ...

// WHEN I'M DONE, I MUST RETURN THE MEMORY!
free(numbers);
```

<Callout type="warning" title="Always use sizeof()">
  Never write `malloc(400)`. Use `malloc(100 * sizeof(int))`. The compiler knows
  exactly how many bytes an `int` occupies on that specific machine (it could be
  2, 4, or 8 bytes).
</Callout>

## Evolution of Strings

In the previous lesson, we saw _Pascal Strings_ (length in one byte, max 255 characters).
Now that we have `malloc`, we can create a serious **String Object**.

Our custom format will be:

1. The first **4 bytes** (`int`) will contain the length of the string.
2. The remaining bytes will contain the characters, plus a final `\0` (to make it compatible with commands like `printf`).

  Unlike standard strings, our PLS (Prefixed Length Strings) can contain null
  bytes (`0x00`) in the middle of the text without "breaking," because we rely
  on the number written in the header instead of searching for a terminator.
</Callout>

### Creating a High-Performance String

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>
#include <string.h>

// Function to create our Super String
char *create_string(char *text) {
int length = strlen(text);

    // Allocate: 4 bytes for the int + length of text + 1 byte for '\\0'
    char *new_str = malloc(sizeof(int) + length + 1);

    if (new_str == NULL) return NULL;

    // 1. Write the length into the first 4 bytes using casting
    *((int *)new_str) = length;

    // 2. Copy the text into the subsequent bytes
    char *text_start = new_str + sizeof(int);
    strcpy(text_start, text);

    return text_start; // Return the pointer to the TEXT, not the length!

}

// Function to instantly read the length in O(1)
int length_v2(char _string_v2) {
// Step back 4 bytes and read the integer
return _((int \*)(string_v2 - sizeof(int)));
}

int main() {
char \*s = create_string("Hello esperto!");

    printf("Text: %s\\n", s);
    printf("Instant length: %d\\n", length_v2(s));

    // Warning: free must receive the original pointer!
    free(s - sizeof(int));

    return 0;

}`}
language="c"
/>
