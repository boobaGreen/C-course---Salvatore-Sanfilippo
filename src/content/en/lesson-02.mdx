## Program Structure and `#include`

Every C program often starts with preprocessor directives. The most common one is `#include <stdio.h>`.

<Callout type="info" title="Standard I/O">
`stdio.h` stands for **Standard Input/Output**. It contains the definitions of fundamental functions like `printf`. Without this include, the compiler wouldn't know how to handle screen output.
</Callout>

### The Compiler is Your Friend: -Wall and -Wextra

Since C gives you a lot of freedom (including the freedom to make mistakes), it's easy to commit logical errors that don't block compilation but cause runtime bugs. For this reason, Salvatore recommends always using **flags** (options) during compilation to enable warnings:

- `-Wall`: Stands for **Warn All**. Despite the name, it enables a vast collection of *common* and fundamental warnings.
- `-Wextra`: Enables even more warnings for subtle and particular cases that `-Wall` doesn't cover.

<Callout type="pro" title="Maximum Security">
Salvatore often uses both: `gcc -Wall -Wextra file.c`. If the compiler "complains," don't ignore it: it's flagging a potential bug before you even run the program.
</Callout>


### The `main` Function and C Constraints
The `main` function is the heart of the program. It is the entry point from which the operating system begins execution. In C, **all executable code must reside within a function**. 

<Callout type="warning" title="No 'Free-Floating' Code">
Unlike Python or Bash, you cannot write a `printf` directly at the top of the file. If you try, the compiler will give you an error like `expected declaration or statement`.
</Callout>

```c
#include <stdio.h>

int main(void) {
    // Your code must stay inside here!
    return 0;
}
```

- `int`: Indicates that the function returns an integer to the operating system (0 means success).
- `void`: Explicitly indicates that the function takes no input arguments.

### A Complete Example: The `sum` Function

Here is the first real program Salvatore showed us. It combines inclusion, function definition, and output:

```c
#include <stdio.h>

// Definition of a function that sums two integers
int sum(int a, int b) {
    return a + b;
}

int main(void) {
    // Call the sum function and print the result
    printf("Hello World %d\n", sum(10, 20));
    return 0;
}
```

Running this code will yield:
`Hello World 30`

## printf: The Magic of Formatting

The name `printf` stands for **Print Formatted**. It is an extremely powerful function because it accepts a **variable number of arguments** (technically called a variadic function).

### Printing Values with Placeholders

To print the result of a function or a variable's value, we must tell `printf` what type of data is expected using **placeholders** (format specifiers) that start with the `%` symbol.

In the `sum` function example, we used `%d`:

```c
printf("Hello World %d\n", sum(10, 20));
```

- `%d`: Stands for **Decimal**. Tells `printf` to expect an integer.
- `\n`: Represents the "newline" character.

### More Parameters, More Power

We can pass many parameters to the same `printf`. It's crucial that the order of the placeholders matches exactly the order of the variables passed after the format string.

```c
printf("Hello %s, result is %d\n", "User", 30);
//      [FORMAT]         [ARG 1]    [ARG 2]
```

<Callout type="danger" title="Warning: Undefined Behavior">
  If you tell `printf` you will print two integers (`%d %d`) but only pass one, C will not stop you! It will read "garbage" from memory or crash. This is your first taste of **Undefined Behavior**.
</Callout>


### The Concept of Expression

Why can we write `sum(10, 20)` inside the `printf`? The answer lies in the **concept of expression**.

In programming, an expression is a piece of code that, once executed by the computer, "becomes a value." This process is called **evaluation**.

- `10 + 20` is an expression evaluated as `30`.
- `sum(10, 20)` is an expression that "calls" the function, executes it, and is replaced by its result (the `return` value), becoming `30`.

When `printf` executes, it no longer sees the function call, but receives the number `30` directly.

## Variables and Explicit Types

In C, unlike languages like Python or JavaScript, you must **explicitly declare** the type of every variable. The compiler does not perform automatic "inference."

```c
int a = 10;   // Declaration and Initialization (assigns 10 during creation)
int b;        // Declaration only (contains RAM "garbage")
b = 20;       // Assignment (overwrites previous value)
int c = a + b; // Initialization via expression
```

<ComparisonTable 
  title="Declaration vs Initialization"
  headers={["Operation", "Example", "Effect in Memory"]}
  rows={[

    ["Initialization", "<code>int x = 10;</code>", "Reserves space and writes the value immediately."],
    ["Assignment", "<code>x = 20;</code>", "Changes the value of an existing space."]
  ]}
/>

<Callout type="pro" title="Expert Mindset">
Specifying the type (e.g., `int`) allows the compiler to know exactly how many bytes to reserve in memory and how to interpret them. This is the secret to C's performance!
</Callout>

## The "Lifecycle" in Memory (The Stack)

What happens when you call a function? Where do local variables end up?
Variables defined inside a function live in a memory area called the **Stack**.

<StackVisualizer title="Stack Concept (Function Call Stack)" />

When a function is called, a **Stack Frame** is created. 

<Callout type="info" title="Creation and Destruction">
  Local variables (including function arguments) are created "on the fly" every time you call the function. When the function returns, the space is released.
</Callout>

<Callout type="warning" title="The Ghost in the Stack">
  **Warning**: "released" does not mean "deleted"! In C, for speed, memory is not wiped when a function ends. Data stays there until another function overwrites it. If you declare `int x;` without initializing it, you'll read the "ghosts" of whoever used that space before.
</Callout>

---

## The `main` Return Value

Why does `main` return an `int`? It's used to tell the **Operating System** if the program finished successfully.
- `return 0;` means "Everything ok".
- `return 1;` (or others) signals an error.

<Callout type="tip">
On Linux, you can see the return value of the last command by typing `echo $?` in the terminal.
</Callout>

### Return Codes and Shell Logic
The return value of `main` is not just a formality. In Unix-like systems, it allows chaining programs based on their success.
- **Logical AND (`&&`)**: `command1 && command2` -> `command2` is executed only if `command1` returns **0** (success).
- **Logical OR (`||`)**: `command1 || command2` -> `command2` is executed only if `command1` fails (non-zero).
