## Variable Scope: Where does your code "live"?

In C, not all variables are created equal. Their "visibility" and "lifetime" depend on where they are declared.

### 1. Local Variables
Defined inside a function. They are born when the function is called and **die** when it returns. Every time you call the function, a new copy is created from scratch.

### 2. Global Variables
Defined outside all functions. They exist for the entire duration of the program and can be accessed from anywhere in the code.

### 3. Static Variables
By using the `static` keyword inside a function, we create a variable that is "global" in duration (it doesn't lose its value between calls) but "local" in visibility (only that function can see it).

## Pass by Value: The Silent Copy

A fundamental concept in C is that function arguments are passed **by value**. This means the function receives a **copy** of the data, not the original.

```c
int incr(int x) {
    x = x + 1;
    return x;
}

int main() {
    int a = 10;
    incr(a); // We pass a COPY of 'a'
    printf("%d", a); // Still prints 10!
}
```

## Numeric Types and Promotions

C offers a variety of types to save memory (crucial back in the '60s!).

### Type Hierarchy
- `char`: 1 byte (8 bits)
- `short`: Typically 2 bytes
- `int`: Typically 4 bytes (implementation-dependent)
- `float`: Single-precision decimal numbers
- `double`: Double-precision decimal numbers (64 bits)

### Automatic Promotions
When performing operations between different types, C applies "promotion" rules. For example, in functions like `printf` (variadic functions):
- `short` values are promoted to `int`.
- `float` values are promoted to `double`.

## Beware of Overflow!

What happens if you add 1 to the maximum possible value for a type?
- **Unsigned**: The behavior is guaranteed. The value "wraps around" back to zero.
- **Signed**: This is **Undefined Behavior**. The compiler can literally do anything. Never rely on signed integer overflow!
