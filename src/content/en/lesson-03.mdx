## Pass by Value: The Silent Copy

A fundamental concept in C is that function arguments are passed **by value**. This means the function receives a **copy** of the data, not the original.

```c
int incr(int x) {
    x = x + 1;
    return x;
}

<Callout type="warning" title="= vs ==">
  Be careful: `=` is the **assignment** operator (moves a value), while `==` is the **comparison** operator. Writing `if (x = 5)` instead of `if (x == 5)` is one of the most common and tricky C errors, because it will assign 5 to x and the condition will always be true!
</Callout>

int main() {
    int a = 10;
    incr(a); // We pass a COPY of 'a'
    printf("%d", a); // Still prints 10!
}
```

## Numeric Types and Promotions

C offers a variety of types to save memory (crucial back in the '60s!).

### Type Hierarchy
- `char`: 1 byte (8 bits)
- `short`: Typically 2 bytes
- `int`: Typically 4 bytes (implementation-dependent)
- `float`: Single-precision decimal numbers
- `double`: Double-precision decimal numbers (64 bits)

### Automatic Promotions
When performing operations between different types, C applies "promotion" rules. This happens because C tries to make calculations smoother (and because in the '60s, it simplified the processor's job).

For example, in functions like `printf` (variadic functions):
- `short` and `char` are promoted to `int`.
- `float` is promoted to `double`.

This means that technically, `printf` doesn't "know" how to print a float: it always receives a double and treats it as such.

<Callout type="info" title="Side Effects">
  A `void` function that takes nothing and returns nothing (like `void hello(void)`) is only useful for its **side effects**: printing to the screen, writing to disk, or turning on an LED. In pure functional programming, these are "sins"; in C, they are daily business.
</Callout>

## Hexadecimal and Advanced Formatting
Did you know you can print a number in base 16 (hexadecimal) just by changing a letter?
- `%d`: Decimal (base 10)
- `%x`: Hexadecimal (base 16)

```c
int a = 10;
printf("Decimal: %d\n", a);   // Prints 10
printf("Hexadecimal: %x\n", a); // Prints a
```

<Callout type="danger" title="Mixing Types is Disastrous">
C is a "dangerous" language: if you tell `printf` to print a float (`%f`) but pass an `int`, the program will read memory incorrectly, printing nonsense values or crashing. The compiler will give you a **warning**, but it will let you compile!
</Callout>

## Beware of Overflow!

What happens if you add 1 to the maximum possible value for a type?
- **Unsigned**: The behavior is guaranteed. The value "wraps around" back to zero.
- **Signed**: This is **Undefined Behavior**. The compiler can literally do anything. Never rely on signed integer overflow!
