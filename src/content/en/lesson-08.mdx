## The Rules of the Game

The Game of Life is a cellular automaton. Each cell can be **ALIVE** (1) or **DEAD** (0). The next state depends only on its 8 neighbors.

<GameOfLife />

<Callout type="info" title="The Toroidal Universe">
  Salvatore uses a brilliant trick: the **modulo** operator (`%`). Instead of
  checking if we've gone off the edges of the array, we use `x % WIDTH`. If we
  reach column 25 in a 25-wide grid, the result is 0. We've "jumped" to the
  other side!
</Callout>

## Problem Decomposition

Salvatore shows us how to "specialize the language" by creating a "ladder" of abstractions. Instead of writing everything in `main`, we create small building blocks:

1.  **`sell_to_index`**: The granular heart. Converts coordinates (x, y) into a linear index, handling wrapping (if I go out at 25, I come back at 0).
2.  **`get_cell` / `set_cell`**: These use `sell_to_index` to read or write memory without worrying about calculations every time.
3.  **`count_living_neighbors`**: Cycles around a cell (8 neighbors) and counts how many are alive. Skips itself (offset 0,0).
4.  **`compute_new_state`**: Applies Conway's rules to decide if a cell will live or die in the next generation.

<Callout type="info" title="Elevating the Language">
  Salvatore explains that in C, the first part of a program often serves to
  "specialize the language": you write low-level functions to create a more
  comfortable environment to solve the actual problem. It's like building your
  own tools before starting to work with wood.
</Callout>

### 1D Mapping: Memory is a Line

Even though we think of the grid as a rectangle, computer RAM is a long linear tape. Salvatore teaches us the magic formula for moving along this tape:
`index = (y * width) + x`

<Callout type="hacker" title="The Pointer 'Cheat'">
  Even though we haven't formally studied them yet, Salvatore must use
  **pointers** (`char *grid`) to pass the grid to functions. Without them, the
  functions would work on a copy of the grid and couldn't modify the original!
</Callout>

---

<KeyConcepts title="Hacker Key Concepts">
  - **Wrapping with %**: Simulating infinite spaces with finite memory. -
  **Functions as Layers**: Building a "ladder" of abstractions to avoid drowning
  in details. - **Escape Sequences**: Using codes like `\033[H` to control the
  terminal (cleaning the screen without `system("clear")`).
</KeyConcepts>
