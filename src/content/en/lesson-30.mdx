## Constraint as Opportunity: 6.75 KB of RAM

The ZX Spectrum used a trick to save memory: its 256x192 resolution is monochromatic (1 bit per pixel), while colors are applied to 8x8 blocks.

<Callout type="pro" title="Quantitative Memory Analysis">
  The exact video memory space calculation is: - **Bitmap**: (256 * 192) / 8
  bits = **6144 bytes** (6 KB) - **Attributes**: (32 * 24) blocks = **768
  bytes** (0.75 KB) - **Total**: 6912 bytes = **6.75 KB**. A modern smartphone
  has millions of times this memory!
</Callout>

### The "Labyrinthine" Memory Map

Unlike modern computers, the Spectrum's video memory is not sequential. If you write to `$4000`, you turn on the top-left pixel. However, `$4001` isn't the pixel below it; it's the pixel to the right. To move down one row, you must perform complex bitwise calculations on the memory address.

## Evolutionary Algorithms: Simulated Annealing

How do you transform a modern photo into a pixel-perfect Spectrum image? Salvatore uses **Simulated Annealing**.

1. **Mutation**: Randomly change a bit in the "fake Spectrum RAM."
2. **Fitness**: Calculate how much the new image resembles the original.
3. **Selection**: If it's better, accept it. If it's worse, accept it only if the "Temperature" is high (to avoid getting stuck).

## Hardware Optimization: Euclidean Distance

To measure the "similarity" between two RGB colors (R1, G1, B1) and (R2, G2, B2), we use 3D geometry.

<Callout type="danger" title="Avoid the Square Root!">
  The distance formula would be: $\sqrt{\Delta R^2 + \Delta G^2 + \Delta B^2}$.
  However, extracting a square root is a heavy operation for the CPU. Salvatore uses only the sum of squares. Since if $a^2 > b^2$ then $a > b$, the result of the comparison remains identical but the code is **much faster**.
</Callout>
