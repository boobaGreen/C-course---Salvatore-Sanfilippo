## Mechanics of Chance: How Does `rand()` Work?

Behind `rand()` typically lies a **Linear Congruential Generator (LCG)**. The formula is:
{'$X_{n+1} = (aX_n + c) \pmod m$'}

<Callout type="pro" title="Why is rand() fast?">
  Because it consists of just one multiplication and one addition. It doesn't
  query the hardware or external sensors. This makes it incredibly fast for
  games but **insecure** for cryptography (a hacker can predict the next number
  if they know the previous ones).
</Callout>

### The Seed and State Memory

Without `srand()`, the generator always starts from $X_0 = 1$. This is why you'll always get the same sequence. `srand(time(NULL))` uses the seconds elapsed since 1970 to change the starting point.

---

## Squashing Chaos: The Modulo Operator

To get a number between 0 and 9, we use `rand() % 10`.

<Callout type="warning" title="Modulo Bias">
  If `RAND_MAX` is not an exact multiple of 10, some numbers will appear
  **slightly** more often than others (e.g., 0 might appear 0.00001% more
  often). In scientific simulations, this is an unacceptable error requiring
  algorithms like "Rejection Sampling."
</Callout>

### Final Summary

Randomness in computers is a useful illusion. Managing the seed and understanding the limits of `rand()` is fundamental to writing correct simulations and fair games.
