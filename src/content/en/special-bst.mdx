## What is a Binary Search Tree?

A **Binary Search Tree** is a data structure where each node has at most two children (left and right). The fundamental property is:
- Values on the **left** are always **smaller** than the current node.
- Values on the **right** are always **larger** than the current node.

## The C Implementation

Salvatore defines the base structure very cleanly:

<CodeEditor 
  initialCode={`struct node {
    int val;
    struct node *left;
    struct node *right;
};

// Creating a new node
struct node *create_node(int val) {
    struct node *n = malloc(sizeof(struct node));
    n->val = val;
    n->left = n->right = NULL;
    return n;
}`}
  language="c"
/>

### Insertion (Iterative)

To insert a value, we "descend" into the tree by comparing the value with each node until we find an empty spot (`NULL`).

<CodeEditor 
  initialCode={`struct node *insert(struct node *root, int val) {
    if (root == NULL) return create_node(val);
    struct node *curr = root;
    while (1) {
        if (val < curr->val) {
            if (curr->left == NULL) {
                curr->left = create_node(val);
                break;
            }
            curr = curr->left;
        } else {
            if (curr->right == NULL) {
                curr->right = create_node(val);
                break;
            }
            curr = curr->right;
        }
    }
    return root;
}`}
  language="c"
/>

## The Elegance of Recursion: Sorted Printing

The most magical moment of the lesson is when Salvatore shows how to print the tree's values from smallest to largest with just a few lines of code. This is called **In-order Traversal**.

<CodeEditor 
  initialCode={`void print_sorted(struct node *root) {
    if (root == NULL) return;
    
    print_sorted(root->left);   // 1. Go left (smaller values)
    printf("%d ", root->val);   // 2. Print current node
    print_sorted(root->right);  // 3. Go right (larger values)
}`}
  language="c"
/>

<Callout type="info" title="The algorithm emerges from the structure">
Salvatore emphasizes that you don't need to learn code by heart: if you understand that small values are on the left, the recursive function "writes itself." This is the beauty of computer science.
</Callout>
