## Anatomy of a Node: The Cost of the Pointer

In a BST, every piece of data is wrapped in a structure. But what does it actually cost in terms of silicon?

<ComparisonTable
  title="Spatial Efficiency (64-bit Architecture)"
  headers={["Element", "Size (Bytes)", "Role"]}
  rows={[
    ["int value", "4", "Actual data"],
    ["struct node *left", "8", "RAM address of lesser child"],
    ["struct node *right", "8", "RAM address of greater child"],
    ["Padding", "4", "8-byte alignment"],
    ["TOTAL", "24", "Cost for 1 integer!"],
  ]}
/>

<Callout type="pro" title="Quantitative Analysis">
  To save 4 bytes of data, we use 24 bytes of RAM (6x overhead). This is the
  price for $O(\log N)$ search instead of $O(N)$. Furthermore, each node is
  allocated separately, causing potential fragmentation and poor utilization of
  the **L1 Cache**.
</Callout>

## The Elegance of Recursion: Sorted Printing

In-order printing is the perfect example of how the data structure dictates the algorithm.

<Infographic
  title="In-order Traversal (Left, Root, Right)"
  description="The three steps of sorted printing."
  items={[
    {
      label: "1. Push Left",
      value: "Descend toward the minimum (CPU Stack grows)",
      color: "blue",
    },
    { label: "2. Print", value: "Extract the central value", color: "purple" },
    {
      label: "3. Move Right",
      value: "Explore greater branches",
      color: "emerald",
    },
  ]}
/>

<Callout type="warning" title="Stack Overflow">
  If the tree is unbalanced (e.g., you insert 1, 2, 3, 4, 5 in order), the tree
  becomes a list $N$ nodes long. Recursion on an unbalanced tree of 1 million
  nodes would cause an immediate stack crash.
</Callout>

### Final Summary

BSTs teach us that algorithmic beauty has a physical cost in memory. Understanding this trade-off is what distinguishes a C programmer from everyone else.
