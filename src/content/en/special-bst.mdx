## What is a Binary Search Tree?

A **Binary Search Tree** is a data structure where each node has at most two children (left and right). The fundamental property is:
- Values on the **left** are always **smaller** than the current node.
- Values on the **right** are always **larger** than the current node.

<Callout type="pro" title="Initialization to NULL">
  Forgetting to initialize `left` and `right` pointers to `NULL` is the most common bug. Without this step, C would use 'garbage' addresses, causing the program to crash as soon as we try to determine if a node has children.
</Callout>

To insert a value, we "descend" into the tree by comparing the value with each node until we find an empty spot (`NULL`).

## The Elegance of Recursion: Sorted Printing

The most magical moment of the lesson is when Salvatore shows how to print the tree's values from smallest to largest with just a few lines of code. This is called **In-order Traversal**.

### Sorted Printing

The real magic happens during printing. By following a specific order, we can display the tree's values perfectly sorted.
