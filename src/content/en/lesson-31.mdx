## The New Role of the Programmer: Technical Audit

Using AI to write C requires a _deeper_ understanding of memory than writing it by hand, because you must be able to spot subtle bugs masked by perfect syntax.

<Callout type="pro" title="Hardware Hallucination">
  AI might suggest non-existent compiler flags or ignore data alignment in
  memory. Your job is the **Audit**: verify that the generated code doesn't
  cause `Undefined Behavior`.
</Callout>

### Profiling: The Truth in Data

Instead of optimizing by intuition, we use the `sample` command (or `gprof`/`perf`).

- **How it works**: The CPU is interrupted thousands of times per second. The tool records where the Instruction Pointer (RIP/EIP) is located.
- **Outcome**: If the processor spends 94% of its time inside `compute_diff`, you've found your "enemy."
