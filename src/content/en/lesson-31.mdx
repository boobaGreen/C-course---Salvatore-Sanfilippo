## The New Role of the Programmer

The arrival of *Large Language Models (LLMs)* and coding agents has radically changed our profession. We no longer have to struggle for five or ten minutes writing boilerplate code; we can delegate the writing process and focus on **architecture** and **correctness**.

<WorkflowDiagram />

<Callout type="info" title="The Importance of Fundamentals">
Why then have we learned raw C so far? Because to guide and *audit* (verify) an AI agent, you must perfectly understand memory, pointers, and machine limits. The AI writes the code, but **you** are responsible for the result.
</Callout>

## Optimization: From Global to Local

Let's return to the program from the last lesson. The problem was slowness: every time we randomly changed 3 bytes, the algorithm recalculated the difference over the **entire image** (49,152 pixels x 3 colors). Ridiculous!

Salvatore asks the AI to change the logic: instead of calculating the global difference, we work on a single **8x8 block at a time**.

<PerformanceChart />

long long compute_diff_global() {
    long long diff = 0;
    for (int i = 0; i < WIDTH * HEIGHT * 3; i++) { ... }
    return diff;
}

// After (with AI help): O(1) independent of image size!
long long compute_block_diff(int start_x, int start_y) {
    long long diff = 0;
    for (int y = start_y; y < start_y + 8; y++) {
        for (int x = start_x; x < start_x + 8; x++) { ... }
    }
    return diff;
}`}

## The Art of Prompting and Dithering

Salvatore notices that restricting updates to isolated blocks creates "blocky artifacts" (similar to JPEG compression) because it ignores the edges.

A true **technical conversation** begins with the AI:
1. Asks to work only on local blocks.
2. Annotates the AI's errors (e.g., the loss of the *dithering* effect).
3. Proposes a solution: update colors by choosing coordinates **unaligned** with the physical 8x8 blocks, allowing overlaps.
4. Corrects the AI regarding the fact that the 2x2 'kernel size' for *dithering* had been lost.

<Callout type="pro" title="Dithering with a 2x2 Kernel">
*Dithering* is the art of placing pixels of two colors next to each other to imitate a third. By calculating the difference (error) considering groups of 2x2 pixels at a time (a kernel), Simulated Annealing understands when two adjacent pixels create a good gradient.
</Callout>
