## Making Decisions: if and else

The heart of programming logic is the crossroads. If X happens, do Y; otherwise, do Z.

<Callout type="info" title="What is True in C?">
In C, there is no native `bool` type (at least in classic versions). For C:
- **0** is False.
- **Anything else** (1, -5, 42) is True.
This is why you can write `if (count)` to check if a counter is non-zero!
</Callout>

## The Power of Braces `{}`

In C, you can open a block of code anywhere, not just after an `if`. Every block creates a new level of existence for variables.

<ScopeExplorer />

<Callout type="warning" title="Ghost Variables">
If you declare `int x = 5;` inside a block, once you exit that block `x` no longer exists for the compiler. Trying to use it will cause an error!
</Callout>

<CodeEditor 
  initialCode={`#include <stdio.h>\n\nint main() {\n    int i = 0;\n    loop:\n    printf("%d ", i);\n    i++;\n    if (i < 5) goto loop;\n    return 0;\n}`}

<Callout type="tip" title="Salvatore's Style: Vertically Short Code">
Salvatore often prefers writing short `if` statements on a single line: `if (x) printf("!");`. Why?
1. **Readability**: It's almost natural language ("If x, print").
2. **Density**: More code fits on one terminal screen, reducing scrolling and helping you keep the program's structure in mind.
Indentation in C is not semantic (unlike Python), but it is an art for the human reader.
</Callout>

## Recursion: Elegance at a Cost
A function can call itself. It's an elegant way to count or solve complex problems, but it has a price: every call takes up space on the **Stack**.

<Callout type="warning" title="Stack Overflow">
  While a `while` loop reuses the same memory, recursion creates a new "Stack Frame" for every call. Salvatore showed us how counting to 10 with recursion can occupy nearly **500 bytes** of RAM instead of just a few bytes! If you overdo it, the program explodes (Stack Overflow).
</Callout>

  - **0 is False**: In C, you don't need a boolean type; zero is false, and everything else is true.
  - **Code Blocks**: Curly braces `{}` don't just group instructions; they create new "worlds" for variables.
  - **Shadowing**: An inner variable can "eclipse" an outer one if they share the same name.
