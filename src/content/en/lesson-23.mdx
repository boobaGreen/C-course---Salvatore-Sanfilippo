## The Stack and ToyFort Logic

In ToyFort, if you want to add 5 and 5, you write `5 5 +`. 
1. `5` is pushed onto the stack.
2. Another `5` is pushed onto the stack.
3. `+` pops the two 5s, adds them, and pushes `10` onto the stack.

## Designing Objects (TFObject)

To represent ToyFort's data (integers, strings, symbols, lists), we use **Unions** and **Reference Counting** learned in previous lessons.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

typedef enum { TF_INT, TF_STR, TF_SYM, TF_LIST } TFType;

// The main object of ToyFort
typedef struct TFObject {
    TFType type;
    int ref_count;
    union {
        long long i;
        struct { char *ptr; size_t len; } s; // Strings and Symbols
        struct { struct TFObject **ele; size_t len; } l; // Lists (Programs)
    };
} TFObject;

// Parser state during code reading
typedef struct TFParser {
    char *prg;              // Full program text
    char *next_token;       // Current reading point
} TFParser;
`} 
  language="c" 
/>

<Callout type="pro" title="Why two asterisks?">
`TFObject **ele` is a pointer to an array of pointers. In C, to have a list that contains objects of different types (heterogeneous), we don't save the objects directly but their addresses. By removing one asterisk (`ele[0]`), we get the pointer to the individual object.
</Callout>

## Flow Control: Stack-based IF-ELSE

Salvatore explains how to implement a conditional using lists as "sub-programs". In ToyFort, the `IF` instruction (or rather `IF-ELSE`) pops three elements from the stack:
1. A list representing the **false** branch.
2. A list representing the **true** branch.
3. A boolean (**condition**).

<Callout type="info" title="Deep Dive: argv[0] and Symbolic Links">
During the lesson, Salvatore goes on a detour about how programs (like `ls` or `vim`) can change their behavior based on the name they are called by (using `argv[0]`). This is possible thanks to symbolic links in Unix and is a powerful technique for system bootstrapping.

### Memory and Recursion
Notice how `TFObject` can contain a list of other `TFObject`s. This recursive structure is fundamental for representing the programs themselves: a ToyFort program is a list of "words," which can themselves be lists (like the branches of an IF).
</Callout>

### Final Summary
We have broken the last barrier between code and data. Function pointers allow us to create flexible systems, universal algorithms, and even emulate object-oriented behavior in pure C. It is the secret that lets libraries like the standard library be so powerful and reusable.
