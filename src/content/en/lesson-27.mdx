## Mapping Symbols: The Function Table

Every command in ToyFort is an "entry" in a table. When the interpreter encounters a symbol, it looks for it in this list and, if found, calls the associated function.

<Callout type="pro" title="The Power of SIMD">
  `memcmp` doesn't compare one byte at a time. Modern CPUs have **SIMD**
  instructions (like AVX or NEON) that can load 256/512-bit registers and
  compare 32 or 64 characters in a single clock cycle. Using standard functions
  means taking advantage of the full power of modern hardware for free.
</Callout>

## String Interning: The Memory Trick

In many interpreters (Python, JavaScript, Lua), instead of comparing string content every time, "String Interning" is used:

1. A global table of all unique strings is created.
2. If two strings have the same memory address, they are guaranteed to be identical.

### Implementation: Robust Comparison

Since our `TF_STR` objects store their own length, the comparison must also handle intermediate null bytes.

```c
int compare_string_objects(TFObject *a, TFObject *b) {
    if (a->s.len != b->s.len) return (a->s.len > b->s.len) ? 1 : -1;
    // Thanks to explicit length, we use memcmp (SIMD optimized)
    return memcmp(a->s.ptr, b->s.ptr, a->s.len);
}
```
