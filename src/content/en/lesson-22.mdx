# Lesson 22: Function Pointers

So far, we have seen pointers that point to data (integers, strings, structures). But in C, code also resides in memory (in the **.text** section) and has an address. A **Function Pointer** allows us to treat code as if it were data.

## The .text Section and RIP/EIP Registers

When you compile your program, functions end up in a memory zone called **.text**, which is marked as **Read-Only** and **Executable**. A function pointer is nothing more than the numeric address of that function's first instruction.

<Callout type="pro" title="What happens in the CPU?">
  When you call a function via a pointer, the CPU performs an **indirect call**.
  Instead of jumping to a fixed address, it reads the address from a register or
  RAM and loads it into the **RIP** (Instruction Pointer) register. This is
  where the magic of polymorphism begins!
</Callout>

## Jump Tables: Pure Speed

Replacing 100 `if-else` statements with an array of function pointers changes the lookup time from linear (O(N)) to constant (O(1)). This is the technique used in kernels, emulators, and graphics engines.

### Example: Array of Functions (Jump Table)

<CodeEditor 
  initialCode={`#include <stdio.h>

void add() { printf("Action 0: Add\\n"); }
void del() { printf("Action 1: Delete\\n"); }
void save() { printf("Action 2: Save\\n"); }

int main() {
// Array of 3 function pointers that return void
void (\*commands[])() = {add, del, save};

    int choice = 1; // Imagine this comes from user input

    if(choice >= 0 && choice < 3) {
        commands[choice](); // Direct O(1) jump!
    }

    return 0;

}`}

## Callbacks: qsort

The standard library's `qsort` function is the perfect example: it knows how to sort everything but delegates the comparison logic to you via a callback.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

// Parameters are void* because qsort is universal
int compare_ints(const void *a, const void _b) {
int arg1 = _(const int*)a;
int arg2 = *(const int\*)b;
return (arg1 > arg2) - (arg1 < arg2);
}

int main() {
int d[] = {10, -5, 100, 2, 0};
qsort(d, 5, sizeof(int), compare_ints);

    for(int i=0; i<5; i++) printf("%d ", d[i]);
    return 0;

}`}

### Final Summary

We have broken the last barrier between code and data. Function pointers allow us to create flexible systems, universal algorithms, and even emulate object-oriented behavior in pure C. It is the secret that lets major libraries be scalable and high-performing.
