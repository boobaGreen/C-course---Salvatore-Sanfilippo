# Lesson 22: Function Pointers

So far, we have seen pointers that point to data (integers, strings, structures). But in C, functions also reside in memory and have an address. A **Function Pointer** allows us to treat code as if it were data, opening the doors to callbacks and advanced abstractions.

## The "Strange" Syntax

Declaring a function pointer can seem intimidating. Salvatore's golden rule is: "Take the prototype, wrap the name in parentheses, and put an asterisk in front of it."

## The Real Power: QSort and Interfaces

The most famous use case is the standard library's `qsort` function. `qsort` knows how to sort anything, but it doesn't know *how* to compare your specific data. To solve this, it asks you for a comparison function pointer.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

// Comparison Callback
int compare(const void *a, const void *b) {
    return (*(int*)a - *(int*)b);
}

int main() {
    int array[] = {7, 1, 9, 3, 5};
    int n = 5;

    qsort(array, n, sizeof(int), compare);

    for(int i=0; i<n; i++) printf("%d ", array[i]);
    return 0;
}`} 
  language="c" 
/>

### Final Summary
We have broken the last barrier between code and data. Function pointers allow us to create flexible systems, universal algorithms, and even emulate object-oriented behavior in pure C. It is the secret that lets libraries like the standard library be so powerful and reusable.
