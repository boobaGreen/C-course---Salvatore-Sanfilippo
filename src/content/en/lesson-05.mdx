## The Truth Behind Characters

When you write `char c = 'A';`, the computer doesn't save the letter "A", but the number **65**. This is the core of the ASCII system.

<ASCIIConverter />

<Callout type="warning" title="Single (') vs Double ('') Quotes">
This is one of the trickiest differences for those coming from Python or JS:
- `'A'` (single quotes) is an **integer number** (65). It takes up 1 byte.
- `"A"` (double quotes) is a **string**. It is an array containing 'A' and an invisible `\0` (zero) character to mark the end. It takes up 2 bytes.
Swapping them will drive the compiler crazy!
</Callout>

## Type Promotion

Did you know that if you pass a `char` to a function with variable arguments (like `printf`), C automatically promotes it to an `int`? This is why `%d` works for printing the numeric code of a character too!

<CodeEditor 
  initialCode={`#include <stdio.h>\n\nint main() {\n    char letter = 'A'; // A single integer\n    printf("Character: %c\\n", letter);\n    printf("ASCII Code: %d\\n", letter);\n    \n    // Variadic promotion: char is passed as int\n    printf("Manual casting: %d\\n", (int)letter);\n    return 0;\n}`}

<KeyConcepts title="Hacker Key Concepts">
  - **Characters are numbers**: Every character (`char`) is actually an 8-bit integer.
  - **ASCII Table**: The universal map that assigns a unique number to every text symbol.
  - **Apostrophes vs Quotes**: `'A'` is a character (a number), `"A"` is a string (a more complex container).
</KeyConcepts>
