# Lesson 29: Makefile, Git, and Project Growth

Up until now, we've been compiling ToyFort with long, manual commands. As the project grows (we're over 400 lines!), managing everything by hand becomes a nightmare. In this lesson, we learn to use the tools of the pros: **Make** and **Git**.

## Why Automate?

Salvatore explains that having to rewrite `cc toyfort.c -Wall -O2 -o toyfort` every time is inefficient. Moreover, during development, it's easy to make changes that "break" the code and want to go back.

### The Makefile: The Compilation Recipe

A `Makefile` is a text file containing "instructions" for compiling the program. It's based on **Targets** and **Prerequisites**.

<CodeEditor 
  initialCode={`# The default target (all) depends on toyfort
all: toyfort

# toyfort depends on the .c source
toyfort: toyfort.c
	cc toyfort.c -Wall -O2 -o toyfort

# Target to clean up generated files
clean:
	rm -f toyfort`}
  language="makefile"
/>

<Callout type="warning" title="The Importance of the Tab">
In Makefiles, command lines **must** start with a real Tab character, not spaces! It's one of the few cases where C (or rather, the tools around it) is picky about whitespace.
</Callout>
