## Using Typedef: More Than Just an Alias

`typedef` does not create a new type but a "nickname" for an existing type. Salvatore highly recommends it to avoid scattering the `struct` keyword everywhere.

```c
typedef struct {
    int x;
    int y;
} Point;

Point p; // Much cleaner than 'struct Point p'
```

### Anonymous Structs and Clarity

Notice that in the `typedef` above, we didn't give the struct a name after the `struct` keyword. This is an **Anonymous Struct**: we only need the alias name `Point` to use the object in the program.

## The File API (stdio.h): The Byte Stream

In C, a file is seen as an infinite **stream** (flow) of bytes. The standard library provides us with a pointer to a special structure called `FILE`.

<Callout type="pro" title="Opaque Types: The Secret of Professional Design">
  The `FILE` type is an **Opaque Type**. You don't know what's inside the `FILE`
  struct (and you aren't allowed to see it!). You only use the pointer and the
  library functions. This allows OS developers to change the internal
  implementation of files (perhaps to make them 10x faster) without your code
  needing to be recompiled. It is the essence of **Encapsulation**.
</Callout>

<Callout type="warning" title="Forgetting fclose()">
  Every open file occupies a **File Descriptor** in the Kernel. If you don't
  call `fclose`, you risk exhausting system resources. Furthermore, data written
  with `fwrite` often sits in a "Write Buffer" in RAM: if the program crashes
  before closing, that data will be lost forever because it never physically
  reached the disk!
</Callout>

## Text vs Binary: Why Binary Wins

When saving a game score, we can write it as text (`"100"`) or as binary (`0x00000064`).

<ComparisonTable
  title="I/O Comparison"
  headers={["Feature", "Text (fprintf)", "Binary (fwrite)"]}
  rows={[
    ["Precision", "Rounding risk (e.g., floats)", "Bit-for-bit (Identical)"],
    [
      "Speed",
      "Slow (must convert numbers to strings)",
      "Blazing fast (direct RAM -> Disk copy)",
    ],
    ["Size", "Variable (depends on digits)", "Fixed (sizeof struct)"],
  ]}
/>

<Callout type="pro" title="Object Persistence">
  Using `fwrite(&player, sizeof(Player), 1, fp)` is the fastest way to save a
  program's state. You are literally "photographing" RAM and writing it to a
  file. When you reopen the file with `fread`, the object is reborn exactly as
  it was.
</Callout>

## Exercise: xdump in Binary

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdint.h>

typedef struct {
uint32_t id;
float score;
} Player;

int main() {
Player p1 = {1, 95.5f};

    // Binary Writing
    FILE *fp = fopen("save.dat", "wb");
    if (fp) {
        fwrite(&p1, sizeof(Player), 1, fp);
        fclose(fp);
        printf("Binary save completed (8 bytes).\\n");
    }

    // Binary Reading
    Player p2;
    fp = fopen("save.dat", "rb");
    if (fp) {
        fread(&p2, sizeof(Player), 1, fp);
        printf("Loaded Player ID: %d, Score: %.1f\\n", p2.id, p2.score);
        fclose(fp);
    }

    return 0;

}`}
