## Puntatori a Puntatori

Anche un puntatore è una variabile, e come tale ha un suo indirizzo di memoria. Possiamo dichiarare un "puntatore a un puntatore":
- `int x = 5;`
- `int *y = &x;` (y punta a x)
- `int **z = &y;` (z punta a y)

<Callout type="info" title="Dimensione Universale">
Tutti i puntatori (a `char`, `int`, o altri puntatori) hanno la **stessa dimensione** (8 byte su sistemi a 64 bit), perché contengono sempre un numero che rappresenta un indirizzo di memoria. La differenza nel tipo (`int*` vs `char*`) serve solo al compilatore per sapere "quanto spazio leggere" quando lo deferenziamo.
</Callout>

### Il Casting: Cambiare Occhiali alla Memoria
Salvatore ci mostra che in C possiamo "mentire" al compilatore tramite il **Casting**. Se abbiamo un array di `char`, possiamo convincere il C che sia un array di `short` (interi a 16 bit).
`short *p = (short *)my_string;`
In questo modo, `p[0]` leggerà **due byte** insieme invece di uno solo. È un'operazione potente ma pericolosa, tipica della programmazione di sistema.

## Stringhe e Puntatori

In C, le stringhe sono array di `char` terminati dal carattere speciale `\0` (null terminator). Possiamo scorrere una stringa incrementando un puntatore finché non incontriamo lo zero.

<Callout type="hacker" title="Aritmetica dei Puntatori">
Quando scrivi `p++` su un `int *p`, l'indirizzo non aumenta di 1, ma di `sizeof(int)` (di solito 4 byte). Il C calcola automaticamente lo "skip" necessario per arrivare all'elemento successivo. 
- `char *p; p++` -> +1 byte
- `short *p; p++` -> +2 byte
- `int *p; p++` -> +4 byte
</Callout>

<CodeEditor 
  initialCode={`#include <stdio.h>

void stampa_stringa(char *p) {
    while (*p != '\\0') {
        putchar(*p); // Stampa il carattere corrente
        p++;         // Passa al prossimo byte
    }
    putchar('\\n');
}

int main() {
    char *saluto = "Hacker World";
    stampa_stringa(saluto);
    return 0;
}`}
  language="c"
/>

<KeyConcepts title="Hacker Key Concepts">
  - **Buffer Overflow**: Cosa succede quando scrivi oltre i limiti di un puntatore/array.
  - **Memory Layout**: Capire come i dati sono impacchettati (punti di forza del C).
  - **Little Endian**: Come i numeri multi-byte sono memorizzati (il byte meno significativo all'indirizzo più basso).

<Callout type="warning" title="L'Ordine dei Byte (Endianness)">
  Se leggi un `int` a 32 bit come se fosse una sequenza di 4 `char`, su sistemi moderni (Intel/Apple) vedrai i byte in ordine "inverso". Il numero `0x12345678` verrà salvato come `78 56 34 12`. Si chiama **Little Endian**.
</Callout>
</KeyConcepts>
