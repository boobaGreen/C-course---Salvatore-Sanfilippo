## La Fisicità della Union

Mentre in una `struct` ogni campo ha il suo indirizzo unico, in una `union` tutti i campi iniziano allo stesso **offset 0**. È come avere una stanza con più etichette sulla porta: il contenuto è uno solo, ma puoi chiamarlo in modi diversi.

## Bitfield: Programmare il Silicio

I Bitfield ti permettono di mappare i singoli bit di un byte a nomi di variabili. Questo è il pane quotidiano di chi scrive **Driver** o **Protocolli di Rete**.

<Callout type="pro" title="Esempio: Registro Hardware">
  Immagina un chip che controlla dei LED. Il suo registro è un byte: i bit 0-3 controllano il colore, il bit 4 è l'accensione.
  ```c
  struct LED_Register {
      unsigned char color  : 4; // solo 4 bit
      unsigned char active : 1; // 1 bit (0 o 1)
      unsigned char unused : 3; // 3 bit di scarto
  };
  ```
</Callout>

### Il Rischio della Portabilità

<Callout type="danger" title="Attenzione all'Endianness dei Bit">
  Non solo i byte possono essere invertiti (Little/Big Endian), ma anche l'ordine dei bit all'interno di un bitfield dipende dal compilatore! Se scambi una struct con bitfield tra un PC x86 e un controller ARM, i bit potrebbero essere letti "al contrario". Per il codice di rete, usa sempre gli operatori bitwise (`&`, `|`, `<<`) che sono universali.
</Callout>

## Esercitazione: Endianness via Union

#include <stdint.h>

typedef union {

ec.val = 0x12345678;

    printf("Dump dell'intero 0x12345678:\\n");

        printf("Byte %d: 0x%02x\\n", i, ec.bytes[i]);

    } else {
        printf("Sistema: BIG ENDIAN\\n");
    }

    return 0;

}`}

### Riassunto Finale

Abbiamo imparato a spremere ogni singolo bit dalla nostra RAM. Le `union` ci permettono di vedere la memoria da angolazioni diverse, mentre i `bitfield` ci danno un controllo chirurgico sullo spazio. Insieme, queste tecniche rendono il C il linguaggio d'elezione per sistemi ad alte prestazioni come Redis e per il controllo diretto dell'Hardware.
