# Lezione 22: Puntatori a Funzioni

Fino ad ora abbiamo visto puntatori che puntano a dati (interi, stringhe, strutture). Ma in C, anche le funzioni risiedono in memoria e hanno un indirizzo. Un **Puntatore a Funzione** ci permette di trattare il codice come se fosse un dato, aprendo le porte a callback e astrazioni avanzate.

## La Sintassi "Strana"

Dichiarare un puntatore a funzione può sembrare intimidatorio. La regola d'oro di Salvatore è: "Prendi il prototipo, avvolgi il nome tra parentesi e mettici un asterisco davanti".

## Il Vero Potere: QSort e Interfacce

Il caso d'uso più celebre è la funzione `qsort` della libreria standard. `qsort` sa come ordinare qualsiasi cosa, ma non sa *come* confrontare i tuoi dati specifici. Per questo, ti chiede un puntatore a funzione di comparazione.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

// Callback di comparazione
int compare(const void *a, const void *b) {
    return (*(int*)a - *(int*)b);
}

int main() {
    int array[] = {7, 1, 9, 3, 5};
    int n = 5;

    qsort(array, n, sizeof(int), compare);

    for(int i=0; i<n; i++) printf("%d ", array[i]);
    return 0;
}`} 
  language="c" 
/>

### Riassunto Finale
Abbiamo rotto l'ultima barriera tra codice e dati. I puntatori a funzione ci permettono di creare sistemi flessibili, algoritmi universali e persino di emulare comportamenti ad oggetti in puro C. È il segreto che permette a librerie come la standard library di essere così potenti e riutilizzabili.
