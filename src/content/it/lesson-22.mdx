# Lezione 22: Puntatori a Funzioni

Fino ad ora abbiamo visto puntatori che puntano a dati (interi, stringhe, strutture). Ma in C, anche il codice risiede in memoria (nella sezione **.text**) e ha un indirizzo. Un **Puntatore a Funzione** ci permette di trattare il codice come se fosse un dato.

## La Sezione .text e il Registro EIP/RIP

Quando compili il tuo programma, le funzioni finiscono in una zona di memoria chiamata **.text**, che è marcata come **Sola Lettura** ed **Eseguibile**. Un puntatore a funzione non è altro che l'indirizzo della prima istruzione di quella funzione.

<Callout type="pro" title="Cosa succede nella CPU?">
  Quando chiami una funzione tramite puntatore, la CPU esegue una chiamata
  **indiretta**. Invece di saltare a un indirizzo fisso, legge l'indirizzo da un
  registro o dalla RAM e lo carica nel registro **RIP** (Instruction Pointer). È
  qui che nasce la magia del polimorfismo!
</Callout>

## Jump Tables: Velocità Pura

Sostituire 100 `if-else` con un array di puntatori a funzione trasforma il tempo di ricerca da lineare (O(N)) a costante (O(1)). È la tecnica usata nei kernel, negli emulatori e nei motori grafici.

### Esempio: Array di Funzioni (Jump Table)

<CodeEditor 
  initialCode={`#include <stdio.h>

void add() { printf("Azione 0: Aggiunta\\n"); }
void del() { printf("Azione 1: Eliminazione\\n"); }
void save() { printf("Azione 2: Salvataggio\\n"); }

int main() {
// Array di 3 puntatori a funzione che non tornano nulla
void (\*comandi[])() = {add, del, save};

    int scelta = 1; // Immagina venga dall'utente

    if(scelta >= 0 && scelta < 3) {
        comandi[scelta](); // Salto diretto O(1)!
    }

    return 0;

}`}

## Callback: qsort

La funzione `qsort` della libreria standard è l'esempio perfetto: sa ordinare tutto, ma delega a te la logica del confronto tramite una callback.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

// I parametri sono void* perché qsort è universale
int compare_ints(const void *a, const void _b) {
int arg1 = _(const int*)a;
int arg2 = *(const int\*)b;
return (arg1 > arg2) - (arg1 < arg2);
}

int main() {
int d[] = {10, -5, 100, 2, 0};
qsort(d, 5, sizeof(int), compare_ints);

    for(int i=0; i<5; i++) printf("%d ", d[i]);
    return 0;

}`}

### Riassunto Finale

Abbiamo rotto l'ultima barriera tra codice e dati. I puntatori a funzione ci permettono di creare sistemi flessibili, algoritmi universali e persino di emulare comportamenti ad oggetti in puro C. È il segreto che permette alle grandi librerie di essere scalabili e performanti.
