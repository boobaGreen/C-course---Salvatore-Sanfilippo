## Perché il Buffering?

Immagina di voler scrivere un milione di caratteri.
1. Se chiami `write()` per ogni carattere, farai 1 milione di syscall (lentissimo).
2. Se usi `fprintf()`, la libreria riempie un buffer di 8KB e chiama `write()` solo quando è pieno (molto più veloce).

<Callout type="info" title="Svuotare il Buffer">
Puoi forzare la scrittura immediata dei dati accumulati nella Libc verso il Kernel usando `fflush(stdout)`. È fondamentale quando vuoi che un messaggio appaia *subito*, senza aspettare il newline o il riempimento del buffer.
</Callout>

### Lo Scheduler e il Multitasking
Salvatore spiega che quando chiami `sleep()`, il tuo processo non "brucia" cicli CPU. Il Kernel lo mette in pausa e dà tempo ad altri processi. Questo è il cuore del **Multitasking Preemptive**.

## mmap: Il File è la tua RAM

Con `mmap`, chiediamo al Kernel di collegare un file direttamente a un indirizzo di memoria del nostro processo. Non serve più `read()` o `write()`: basta leggere dal puntatore!

### Esempio: Leggere un file con mmap

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd = open("test.txt", O_RDONLY);
    struct stat st;
    fstat(fd, &st); // Troviamo la dimensione del file

    // Mappiamo il file in memoria sola lettura
    char *data = mmap(NULL, st.st_size, PROT_READ, MAP_PRIVATE, fd, 0);

    if (data == MAP_FAILED) {
        perror("mmap fallito");
        return 1;
    }

    // Ora possiamo leggere il file come se fosse una stringa!
    printf("Contenuto file: %s\\n", data);

    munmap(data, st.st_size);
    close(fd);
    return 0;
}`} 
  language="c" 
/>

<Callout type="hacker" title="MAP_FAILED">
Attenzione: `mmap` non ritorna `NULL` in caso di errore, ma una costante chiamata `MAP_FAILED` (che di solito corrisponde all'indirizzo `0xFFFFFFFFFFFFFFFF`). Controlla sempre questo valore per evitare crash misteriosi!
</Callout>

### Perché mmap è "Magica"?
Elimina il passaggio di copia dei dati tra lo spazio Kernel e lo spazio Utente. Il file *diventa* la tua memoria. È la tecnica usata dai database più veloci (come Redis o MongoDB) per gestire enormi dataset con la massima efficienza.
