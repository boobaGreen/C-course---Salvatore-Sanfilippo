## Perché il Buffering? La Fisica della Velocità

Andare nel Kernel (Ring 0) è come passare attraverso una dogana: devi fermarti, mostrare i documenti e aspettare. Se lo fai per ogni singolo carattere, il programma rallenta di 100 volte.

## mmap: La Tecnica Suprema

Con `mmap`, chiediamo al Kernel di collegare un file direttamente a un indirizzo di memoria del nostro processo. Non serve più `read()` o `write()`: basta leggere dal puntatore!

<Callout type="pro" title="Zero-Copy Architecture">
  Normalmente `read()` copia i dati dal disco al Kernel, poi dal Kernel al tuo
  buffer in RAM. `mmap` elimina questa seconda copia. Il tuo puntatore punta
  direttamente alle pagine di memoria gestite dal Kernel (**Page Cache**). È il
  segreto della velocità di Redis e dei moderni motori di database.
</Callout>

### La Magia del Page Fault

Quando mappi un file di 1GB, il Kernel non lo carica tutto subito. Crea solo una "mappa". Quando il tuo codice tocca un indirizzo in quel giga, l'Hardware genera un **Page Fault**. Il Kernel si sveglia, carica solo quella pagina (4KB) dal disco e ridà il controllo al tuo programma. È un'illusione perfetta di memoria infinita e immediata.

## Implementazione: Accesso Diretto ai File

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <sys/mman.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>

int main() {
int fd = open("test.txt", O_RDONLY);
if (fd == -1) {
printf("Crea un file test.txt per provare!\\n");
return 1;
}

    struct stat st;
    fstat(fd, &st); // Dimensione del file dal Kernel

    // MAP_PRIVATE: le modifiche restano in RAM (Copy-on-Write)
    // PROT_READ: sola lettura
    char *data = mmap(NULL, st.st_size, PROT_READ, MAP_PRIVATE, fd, 0);

    if (data == MAP_FAILED) {
        perror("mmap fallito");
        return 1;
    }

    // Il file ora è un array!
    for (int i = 0; i < st.st_size; i++) {
        putchar(data[i]);
    }

    munmap(data, st.st_size); // Sconnetti la memoria
    close(fd);
    return 0;

}`}
