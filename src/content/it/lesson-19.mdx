## File Descriptors: 0, 1, 2

Ogni processo Linux nasce con tre canali di comunicazione già aperti:
- **0**: Standard Input (tastiera)
- **1**: Standard Output (terminale)
- **2**: Standard Error (errori sul terminale)

<Callout type="info" title="Perché i numeri?">
Il Kernel è pigro (o meglio, efficiente). Non vuole gestire stringhe lunghe per ogni file. Usa un semplice array per ogni processo, e l'indice di quell'array è il nostro **File Descriptor**.

<Callout type="hacker" title="Tutto è un File">
  Questa è la filosofia zen di Unix: file su disco, tastiera, monitor e persino la scheda di rete vengono trattati tutti come **File Descriptors**. Puoi usare `write()` per salvare un documento o per inviare un messaggio su internet: per il programmatore C, la logica è la stessa.
</Callout>

## Esempio: Scrittura a Basso Livello

Usiamo `open` e `write` invece delle funzioni standard. Nota come dobbiamo gestire i permessi ottali (es. 0644).

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <unistd.h>
#include <fcntl.h>
#include <errno.h>
#include <string.h>

int main() {
    // Apriamo un file in sola scrittura, creandolo se manca
    int fd = open("low_level.txt", O_WRONLY | O_CREAT | O_TRUNC, 0644);
    
    if (fd == -1) {
        perror("Errore nell'apertura");
        return 1;
    }

    char *msg = "Saluti dal Kernel!\\n";
    if (write(fd, msg, strlen(msg)) == -1) {
        perror("Errore nella scrittura");
    }

    close(fd);
    return 0;
}`} 
  language="c" 
/>

<Callout type="hacker" title="Ancestrale ma Attuale">
Le system calls che vedi qui (come `open`) sono state definite negli anni '70 per Unix. Cinquant'anni dopo, alimentano ancora Linux, macOS e quasi tutto il cloud mondiale. Impararle significa connettersi ai "pionieri" dell'informatica.
</Callout>

### errno e perror
Quando una syscall fallisce (ritorna -1), il Kernel imposta un numero globale chiamato `errno`. La funzione `perror` legge quel numero e lo traduce in un messaggio umano (es. "No such file or directory").

<Callout type="warning" title="Costo delle Syscalls">
  Chiamare una syscall è "costoso". La CPU deve fermare il tuo programma, cambiare modalità (da User a Kernel), fare il lavoro e tornare indietro. È per questo che `printf` usa un buffer: accumula i dati e fa una sola `write()` finale invece di cento piccole chiamate, risparmiando tempo prezioso.
</Callout>
