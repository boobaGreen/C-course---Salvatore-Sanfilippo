## Il "Hello World" in C

Il C è un linguaggio molto vecchio che si è evoluto relativamente poco nella sintassi base, ma moltissimo nei **compilatori**. Rimane un linguaggio fatto di "poche idee", che vanno però assimilate a fondo. Essendo un **linguaggio compilato**, non possiamo eseguire direttamente il codice sorgente: dobbiamo prima tradurlo in linguaggio macchina.

Ecco il classico programma di partenza, in sole 5 righe:

```c
#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
```

## Dietro le quinte: l'Assembly e le Ottimizzazioni

Possiamo chiedere al compilatore di fermarsi prima di generare il binario finale, facendoci mostrare il codice **Assembly** intermedio, usando il flag `-S`:

<TerminalSimulation 
  command="cc -S hello.c" 
  language="asm"
>
{`	.file	"hello.c"
	.text
	.section	.rodata
.LC0:
	.string	"Hello, World!"
	.text
	.globl	main
	.type	main, @function
main:
.LFB0:
	.cfi_startproc
	pushq	%rbp
	.cfi_def_cfa_offset 16
	.cfi_offset 6, -16
	movq	%rsp, %rbp
	.cfi_def_cfa_register 6
	leaq	.LC0(%rip), %rdi
	call	puts@PLT
	movl	$0, %eax
	popq	%rbp
	.cfi_def_cfa 7, 8
	ret
	.cfi_endproc`}
</TerminalSimulation>

Guardando l'Assembly, possiamo notare che viene definita la stringa "Hello, World!" e viene chiamata la funzione `printf`. Il bello del C moderno sta però nei **livelli di ottimizzazione**. 

Se compiliamo con il flag `-O2` (ottimizzazione di livello 2):

<TerminalSimulation 
  command="cc -O2 -S hello.c" 
  language="asm"
>
{`	.file	"hello.c"
	.text
	.section	.rodata.str1.1,"aMS",@progbits,1
.LC0:
	.string	"Hello, World!"
	.section	.text.startup,"ax",@progbits
	.p2align 4
	.globl	main
	.type	main, @function
main:
.LFB23:
	.cfi_startproc
	subq	$8, %rsp
	.cfi_def_cfa_offset 16
	leaq	.LC0(%rip), %rdi
	call	puts@PLT
	xorl	%eax, %eax
	addq	$8, %rsp
	.cfi_def_cfa_offset 8
	ret
	.cfi_endproc`}
</TerminalSimulation>

Noteremo una cosa sorprendente: **la funzione `printf` scompare**! Il compilatore, accorgendosi che stiamo stampando una stringa semplice seguita da un ritorno a capo (`\n`), sostituisce automaticamente la `printf` con la funzione `puts`, che è più veloce ed efficiente per questo scopo specifico, rimuovendo il `\n` originale perché `puts` lo aggiunge da sola.
Questa è solo una "goccia nell'oceano" delle ottimizzazioni che un compilatore maturo può fare.

## Facciamo a pezzi il programma

Analizziamo ogni riga per capire cosa succede realmente:

### 1. Il Preprocessore e `#include`
```c
#include <stdio.h>
```
Questa riga non fa parte del linguaggio C vero e proprio, ma è una **direttiva per il preprocessore** (tutte le direttive iniziano con il `#`). Prima della compilazione vera e propria, il preprocessore prende il contenuto del file `stdio.h` (un file di libreria standard) e lo "scompatta" direttamente all'interno del nostro file. 

Perché ne abbiamo bisogno? Il linguaggio C in sé ha pochissime parole chiave (`return`, `int`, `while`). Funzioni come `printf` non fanno parte del linguaggio, ma della **Libreria Standard**. Se non includessimo `stdio.h` (Standard Input/Output Header), il compilatore non saprebbe cosa sia `printf`!

### 2. File Header (`.h`) e Prototipi di Funzione

Nei file header (come `stdio.h`) non c'è il codice che *esegue* la `printf`, ma ci sono i **prototipi di funzione**.

Un prototipo è simile alla definizione di una funzione, ma invece di avere un blocco di codice tra parentesi graffe `{}`, termina subito con un punto e virgola `;`:

```c
int printf(const char *format, ...);
```

Il prototipo serve a dire al compilatore: *"Esiste una funzione chiamata `printf` che accetta questi parametri e restituisce un numero intero. Fidati, la troverai più tardi"*. 

Grazie al prototipo, il compilatore sa esattamente come preparare i dati in memoria (Assembly) per chimarla correttamente, anche senza conoscerne il codice interno. Infatti, potremmo anche non usare `#include <stdio.h>` e scrivere a mano il prototipo nel nostro file per far funzionare il programma!

<KeyConcepts title="Ricapitolando">
  - Il C non è eseguito, ma **compilato**.
  - Il compilatore sa ottimizzare il codice in modo intelligente (es. trasformando `printf` in `puts`).
  - Il **preprocessore** prepara il file prima della compilazione.
  - I file **Header (`.h`)** contengono i **prototipi** delle funzioni, essenziali per dire al compilatore come chiamare funzioni scritte altrove.
</KeyConcepts>
