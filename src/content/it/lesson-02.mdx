# Lezione 2: Variabili, Funzioni e Memoria

## Struttura del Programma e `#include`

Ogni programma C inizia spesso con delle direttive al preprocessore. La più comune è `#include <stdio.h>`.

<Callout type="info" title="Standard I/O">
  `stdio.h` sta per **Standard Input/Output**. Contiene le definizioni di
  funzioni fondamentali come `printf`. Senza questo include, il compilatore non
  saprebbe come gestire l'output a video.
</Callout>

### Il Compilatore ti aiuta: -Wall e -Wextra

Dato che il C ti lascia molta libertà (anche quella di sbagliare), è facile commettere errori logici che non bloccano la compilazione ma causano bug a runtime. Per questo motivo, Salvatore consiglia di usare sempre dei **flag** (opzioni) durante la compilazione per attivare gli avvisi:

- `-Wall`: Sta per **Warn All**. Nonostante il nome, abilita una vasta collezione di avvisi _comuni_ e fondamentali.
- `-Wextra`: Abilita ancora più avvisi per casi particolari e sottili che `-Wall` non copre.

<Callout type="pro" title="Massima Sicurezza">
  Salvatore li usa spesso entrambi: `gcc -Wall -Wextra file.c`. Se il
  compilatore "si lamenta", non ignorarlo: ti sta segnalando un potenziale bug
  prima ancora che tu esegua il programma.
</Callout>

### La Funzione `main` e i Vincoli del C

La funzione `main` è il cuore del programma. È il punto di ingresso (entry point) da cui il sistema operativo inizia l'esecuzione. In C, **tutto il codice eseguibile deve trovarsi all'interno di una funzione**.

<Callout type="warning" title="Niente Codice 'Volante'">
  A differenza di Python o Bash, non puoi scrivere una `printf` direttamente
  all'inizio del file. Se provi a farlo, il compilatore ti darà un errore del
  tipo `expected declaration or statement`.
</Callout>

```c
#include <stdio.h>

int main(void) {
    // Il tuo codice deve stare qui dentro!
    return 0;
}
```

- `int`: Indica che la funzione restituisce un numero intero al sistema operativo (0 significa successo).
- `void`: Indica esplicitamente che la funzione non accetta argomenti in ingresso.

### Un Esempio Completo: La funzione `sum`

Ecco il primo vero programma che Salvatore ci ha mostrato. Combina inclusione, definizione di una funzione e output:

```c
#include <stdio.h>

// Definizione di una funzione che somma due interi
int sum(int a, int b) {
    return a + b;
}

int main(void) {
    // Chiamiamo la funzione sum e stampiamo il risultato
    printf("Hello World %d\n", sum(10, 20));
    return 0;
}
```

Eseguendo questo codice, otterrai:
`Hello World 30`

## printf: La Magia della Formattazione

Il nome `printf` sta per **Print Formatted**. È una funzione estremamente potente perché accetta un **numero variabile di argomenti** (tecnicamente detta funzione variadica).

### Stampare Valori con i Segnaposto

Per stampare il risultato di una funzione o il valore di una variabile, dobbiamo dire alla `printf` che tipo di dato si aspetta usando dei **segnaposto** (format specifier) che iniziano con il simbolo `%`.

Nell'esempio della funzione `sum`, abbiamo usato `%d`:

```c
printf("Hello World %d\n", sum(10, 20));
```

- `%d`: Sta per **Decimal**. Dice alla `printf` di aspettarsi un numero intero.
- `\n`: Rappresenta il carattere di "nuova riga".

### Più Parametri, Più Potere

Possiamo passare molti parametri alla stessa `printf`. L'importante è che l'ordine dei segnaposto corrisponda esattamente all'ordine delle variabili passate dopo la stringa di formato.

```c
printf("Ciao %s, il risultato è %d\n", "User", 30);
//      [FORMAT]           [ARG 1]     [ARG 2]
```

<Callout type="danger" title="Attenzione: Undefined Behavior">
  Se dici alla `printf` che stamperai due interi (`%d %d`) ma gliene passi solo
  uno, il C non si fermerà! Leggerà "immondizia" dalla memoria o andrà in crash.
  Questo è il primo assaggio della cosiddetta **Undefined Behavior**
  (Comportamento Indefinito).
</Callout>

### Il Concetto di Espressione

Perché possiamo scrivere `sum(10, 20)` dentro la `printf`? La risposta sta nel **concetto di espressione**.

In programmazione, un'espressione è un pezzo di codice che, una volta eseguito dal computer, "diventa un valore". Questo processo si chiama **valutazione**.

- `10 + 20` è un'espressione che viene valutata come `30`.
- `sum(10, 20)` è un'espressione che "chiama" la funzione, la esegue e alla fine viene sostituita dal suo risultato (il valore di `return`), diventando `30`.

Quando la `printf` viene eseguita, non vede più la chiamata alla funzione, ma riceve direttamente il numero `30`.

## Variabili e Tipi Espliciti

In C, a differenza di linguaggi come Python o JavaScript, devi **dichiarare esplicitamente** il tipo di ogni variabile. Il compilatore non fa "inferenza" automatica.

```c
int a = 10;   // Dichiarazione e Inizializzazione (assegna 10 durante la creazione)
int b;        // Sola Dichiarazione (contiene "spazzatura" della RAM)
b = 20;       // Assegnazione (sovrascrive il valore precedente)
int c = a + b; // Inizializzazione tramite espressione
```

<Callout type="pro" title="Expert Mindset">

  quanti byte riservare in memoria e come interpretarli. Questo è il segreto
  delle performance del C!
</Callout>

## Il "Ciclo di Vita" nella Memoria (Lo Stack)

Cosa succede quando chiami una funzione? Dove finiscono le variabili locali?
Le variabili definite dentro una funzione vivono in una zona di memoria chiamata **Stack** (Pila).

<StackVisualizer title="Concetto di Stack (Pila delle Funzioni)" />

Quando una funzione viene chiamata, viene creato un **Stack Frame**.

<Callout type="info" title="Creazione e Distruzione">
  Le variabili locali (inclusi gli argomenti della funzione) vengono create "al
  volo" ogni volta che chiami la funzione. Quando la funzione ritorna con
  `return`, lo spazio viene liberato.
</Callout>

<Callout type="warning" title="The Ghost in the Stack">
  **Attenzione**: "liberato" non significa "cancellato"! In C, per motivi di
  performance, la memoria non viene azzerata quando una funzione termina. I dati
  rimangono lì finché un'altra funzione non sovrascrive quel pezzetto di Stack.
  Se dichiari `int x;` senza inizializzarla, leggerai i "fantasmi" di chi ha
  usato quel posto prima di te.
</Callout>

## Il valore di ritorno di `main`

Perché `main` ritorna un `int`? Serve a comunicare al **Sistema Operativo** se il programma è terminato correttamente.

- `return 0;` significa "Tutto ok".
- `return 1;` (o altro) segnala un errore.

<Callout type="tip">
  Su Linux, puoi vedere il valore di ritorno dell'ultimo comando digitando `echo
  $?` nel terminale.
</Callout>

### Codici di Ritorno e Logica Shell

Il valore di ritorno di `main` non è solo una formalità. In sistemi Unix, permette di concatenare programmi in base al loro successo.

- **AND Logico (`&&`)**: `comando1 && comando2` -> `comando2` viene eseguito solo se `comando1` ritorna **0** (successo).
- **OR Logico (`||`)**: `comando1 || comando2` -> `comando2` viene eseguito solo se `comando1` fallisce (diverso da 0).
