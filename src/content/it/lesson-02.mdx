# Lezione 2: Variabili, Funzioni e Memoria

Benvenuti alla seconda lezione! Oggi andremo a fondo nella struttura di un programma C, capendo finalmente cosa succede "sotto il cofano" quando dichiariamo una variabile o chiamiamo una funzione.

## Struttura del Programma e `#include`

Ogni programma C inizia spesso con delle direttive al preprocessore. La più comune è `#include <stdio.h>`.

<Callout type="info" title="Standard I/O">
`stdio.h` sta per **Standard Input/Output**. Contiene le definizioni di funzioni fondamentali come `printf`. Senza questo include, il compilatore non saprebbe come gestire l'output a video.
</Callout>

### La Funzione `main` e i Vincoli del C
La funzione `main` è il cuore del programma. È il punto di ingresso (entry point) da cui il sistema operativo inizia l'esecuzione. In C, **tutto il codice eseguibile deve trovarsi all'interno di una funzione**. 

<Callout type="warning" title="Niente Codice 'Volante'">
A differenza di Python o Bash, non puoi scrivere una `printf` direttamente all'inizio del file. Se provi a farlo, il compilatore ti darà un errore del tipo `expected declaration or statement`.
</Callout>

```c
int main(void) {
    // Il tuo codice deve stare qui dentro!
    return 0;
}
```

- `int`: Indica che la funzione restituisce un numero intero al sistema operativo (0 significa successo).
- `void`: Indica esplicitamente che la funzione non accetta argomenti in ingresso.

---

## printf: La Magia della Formattazione

Il nome `printf` sta per **Print Formatted**. È una funzione estremamente potente perché è **variadica**: può accettare un numero variabile di argomenti.

<Callout type="danger" title="Pericoli Variadici">
Se dici alla `printf` che stamperai due interi (`%d %d`) ma gliene passi solo uno, il C non si fermerà! Leggerà "immondizia" dalla memoria o andrà in crash. Questo è il primo assaggio della cosiddetta **Undefined Behavior** (Comportamento Indefinito).
</Callout>

```c
printf("Ciao %s, il risultato è %d\n", "User", 30);
//      [FORMAT]           [ARG 1]     [ARG 2]
```

## Variabili e Tipi Espliciti

In C, a differenza di linguaggi come Python o JavaScript, devi **dichiarare esplicitamente** il tipo di ogni variabile. Il compilatore non fa "inferenza" automatica.

```c
int a = 10;   // Dichiarazione e Inizializzazione (assegna 10 durante la creazione)
int b;        // Sola Dichiarazione (contiene "spazzatura" della RAM)
b = 20;       // Assegnazione (sovrascrive il valore precedente)
int c = a + b; // Inizializzazione tramite espressione
```

<Callout type="pro" title="Expert Mindset">
Specificare il tipo (es. `int`) permette al compilatore di sapere esattamente quanti byte riservare in memoria e come interpretarli. Questo è il segreto delle performance del C!
</Callout>

## Il "Ciclo di Vita" nella Memoria (Lo Stack)

Cosa succede quando chiami una funzione? Dove finiscono le variabili locali?
Le variabili definite dentro una funzione vivono in una zona di memoria chiamata **Stack** (Pila).

<StackVisualizer />

Quando una funzione viene chiamata, viene creato un **Stack Frame**. 

<Callout type="info" title="Creazione e Distruzione">
  Le variabili locali (inclusi gli argomenti della funzione) vengono create "al volo" ogni volta che chiami la funzione. Quando la funzione ritorna con `return`, lo spazio viene liberato. 
</Callout>

<Callout type="warning" title="The Ghost in the Stack">
  **Attenzione**: "liberato" non significa "cancellato"! In C, per motivi di performance, la memoria non viene azzerata quando una funzione termina. I dati rimangono lì finché un'altra funzione non sovrascrive quel pezzetto di Stack. Se dichiari `int x;` senza inizializzarla, leggerai i "fantasmi" di chi ha usato quel posto prima di te.
</Callout>

Quando la funzione termina (`return`), il suo frame viene "distrutto" e la memoria liberata.

## Il valore di ritorno di `main`

Perché `main` ritorna un `int`? Serve a comunicare al **Sistema Operativo** se il programma è terminato correttamente.
- `return 0;` significa "Tutto ok".
- `return 1;` (o altro) segnala un errore.

<Callout type="tip">
Su Linux, puoi vedere il valore di ritorno dell'ultimo comando digitando `echo $?` nel terminale.
</Callout>

### Codici di Ritorno e Logica Shell
Il valore di ritorno di `main` non è solo una formalità. In sistemi Unix, permette di concatenare programmi in base al loro successo.
- **AND Logico (`&&`)**: `comando1 && comando2` -> `comando2` viene eseguito solo se `comando1` ritorna **0** (successo).
- **OR Logico (`||`)**: `comando1 || comando2` -> `comando2` viene eseguito solo se `comando1` fallisce (diverso da 0).

Questo meccanismo di "short-circuit" è fondamentale per automatizzare task complessi dove ogni passo dipende dal precedente.

<Callout type="tip">
I **Registri** sono piccole "variabili" dentro la CPU, velocissime. La **RAM** è più grande ma più lenta. Il compilatore C cerca di usare i registri il più possibile per velocizzare il tuo codice.
</Callout>
