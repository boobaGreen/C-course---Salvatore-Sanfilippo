# Lezione 13: Stringhe di Alto Livello e Metadati Nascosti

Benvenuti ad una delle lezioni più cruciali del corso. Oggi impareremo a "scolpire" la memoria per creare oggetti intelligenti. A differenza delle stringhe standard, le nostre PLS (Prefixed Length Strings) possono contenere byte a zero (`0x00`) nel mezzo del testo senza "rompersi", perché ci affidiamo al numero scritto nell'header invece di cercare il terminatore.

<Callout type="warning" title="Sicurezza dei Metadati">
  Salvando la lunghezza *prima* del puntatore, proteggiamo il nostro sistema. Se un utente malintenzionato prova a inviare una stringa troppo lunga tramite `strcpy`, sovrascriverà i dati *dopo* la stringa, ma non toccherà la lunghezza (che sta *prima*), rendendo più difficile corrompere la struttura logica del programma.
</Callout>

## Il Problema delle Stringhe Pascal

Nella scorsa lezione abbiamo visto le stringhe a lunghezza prefissa. Funzionavano, ma avevano un difetto fatale: se passavi una stringa Pascal a `printf`, lei "impazziva" perché il primo byte (la lunghezza) veniva interpretato come un carattere. 

Salvatore ci insegna il **trucco del veterano**: spostare il prefisso **prima** dell'indirizzo di memoria che diamo all'utente.

### Perché `uint32_t` di `<stdint.h>`?

Salvatore è categorico: non possiamo usare un semplice `int`. 
Un `int` su un sistema a 16 bit occupa 2 byte, su uno a 32 bit ne occupa 4. Se vogliamo che le nostre stringhe siano portabili e che il nostro header occupi **sempre** lo stesso spazio, dobbiamo usare i tipi a dimensione fissa.

## Implementazione della Libreria `ps_string`

Ecco il codice completo discusso da Salvatore, con la logica di **creazione**, **accesso O(1)** e **liberazione corretta**.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <stdint.h>

// Definiamo un header da 4 byte (uint32_t)
#define HDR_SIZE sizeof(uint32_t)

// Crea una stringa evoluta da una stringa C
char *ps_create(char *init) {
    uint32_t len = strlen(init);
    // Allochiamo: Lunghezza (4) + Testo + '\\0' (1)
    char *p = malloc(HDR_SIZE + len + 1);
    if (!p) return NULL;

    // Scriviamo la lunghezza all'inizio
    *((uint32_t*)p) = len;

    // Puntiamo al testo (saltiamo l'header)
    char *s = p + HDR_SIZE;
    // Copiamo il testo (inclusa la compatibilità C)
    strcpy(s, init);
    
    return s;
}

// Ritorna la lunghezza in tempo COSTANTE O(1)
uint32_t ps_len(char *s) {
    // Torniamo indietro di 4 byte e leggiamo il valore
    return *((uint32_t*)(s - HDR_SIZE));
}

// Libera la memoria correttamente
void ps_free(char *s) {
    // CRITICO: Dobbiamo liberare dall'indirizzo ORIGINALE (p)
    free(s - HDR_SIZE);
}

int main() {
    char *my_str = ps_create("Salvatore C Course");
    
    // Interoperabilità: printf vede una normale stringa C!
    printf("Testo: %s\\n", my_str);
    
    // Potenza: Sappiamo la lunghezza senza scansionare la stringa
    printf("Lunghezza (O(1)): %u\\n", ps_len(my_str));
    
    ps_free(my_str);
    return 0;
}`} 
  language="c" 
/>

<Callout type="hacker" title="Reference Counting">
Salvatore introduce una tecnica fondamentale per gestire la memoria condivisa: il **Reference Counting**. Invece di duplicare una stringa ogni volta che un nuovo puntatore la referenzia, incrementiamo un contatore nell'header.
- Quando aggiungi un riferimento: `count++`
- Quando rimuovi un riferimento: `count--`
- Quando `count == 0`: chiami veramente la `free()`.
</Callout>

### Verso le Struct
Manipolare manualmente gli offset (come `s - 4` o `s - 8`) è "fragile" e prono a errori. Per questo Salvatore introduce le **Struct**: un modo per raggruppare dati diversi (lunghezza, refcount, testo) in un'unica entità logica.
```c
struct ps_string {
    uint32_t len;
    uint32_t ref_count;
    char buf[]; // Flexible Array Member (C99)
};
```

## Simulazione Performance: O(N) vs O(1)

Immagina una stringa lunga 10 milioni di caratteri. 
- `strlen` (Standard C) deve leggere **10.000.000 di byte** per trovare lo zero finale.
- `ps_len` (Nostra) deve leggere **solo 1 numero** a un offset fisso.

**Differenza di tempo:** Da millisecondi a nanosecondi. Su larga scala, questo cambia tutto.
