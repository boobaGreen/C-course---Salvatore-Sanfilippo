## Implementare xdump

Salvatore ci mostra come trasformare un puntatore `void *` in una serie di byte leggibili. L'obiettivo è stampare sia il valore esadecimale che il carattere ASCII (se stampabile).

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <string.h>
#include <ctype.h>

void xdump(void *p, int len) {
    unsigned char *c = p;
    for (int i = 0; i < len; i++) {
        // Operatore ternario: se stampabile mostra il char, altrimenti un punto
        printf("%02x (%c) ", c[i], isprint(c[i]) ? c[i] : '.');
        if ((i + 1) % 8 == 0) printf("\\n");
    }
    printf("\\n");
}

## L'Operatore Ternario: `? :`

È la versione "hacker" dell'if-else. Salvatore lo usa spesso per decisioni rapide:
`condizione ? valore_se_vero : valore_se_falso`

Esempio:
`char display = isprint(c) ? c : '.';`

## Memset: Pulizia Profonda

In C, la memoria appena allocata o dichiarata sullo stack contiene "spazzatura". Con `memset` possiamo azzerare tutto in un colpo solo.

<Callout type="info" title="Ottimizzazione di memset">
  Il compilatore è così intelligente che se vede un `memset` seguito immediatamente da una `free`, potrebbe decidere di **non eseguirlo** (perché "non serve" a nulla per la logica del programma). Questo è un problema se stai azzerando una password in memoria per sicurezza! In quei casi si usano funzioni speciali come `explicit_bzero`.
</Callout>

### Riassunto Finale
Abbiamo imparato a non fidarci delle astrazioni: la memoria è un nastro di byte e noi abbiamo ora gli strumenti per ispezionarla e manipolarla con precisione chirurgica. `memset` e `xdump` saranno i tuoi migliori amici nel debugging di basso livello.
