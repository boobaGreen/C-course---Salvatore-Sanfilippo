## Makefile: La Fisica del Tempo

Un `Makefile` non è solo uno script. È un grafo di dipendenze. `make` confronta la data di modifica (**mtime**) del file sorgente con quella del file binario.

<Callout type="pro" title="Logica Hardware di Make">
  Se `toyfort.c` ha un timestamp più recente di `toyfort` (il binario), `make`
  lancia il comando di compilazione. Se il binario è più recente, `make` non fa
  nulla. Questo risparmia ore di lavoro in progetti con migliaia di file.
</Callout>

### Ottimizzazioni in Produzione

Usiamo il flag `-O2`. Questo non solo attiva ottimizzazioni matematiche, ma riorganizza il codice per favorire la **Pipeline della CPU** e la **Branch Prediction**.

Git permette di lavorare senza paura. Ogni `commit` è un'ancora di salvezza hardware salvata nel database di Git.

### Riassunto Finale
