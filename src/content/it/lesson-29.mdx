# Lezione 29: Makefile, Git e la Crescita del Progetto

Fino ad ora abbiamo compilato ToyFort con comandi lunghi e manuali. Man mano che il progetto cresce (siamo oltre le 400 righe!), gestire tutto a mano diventa un incubo. In questa lezione impariamo a usare gli strumenti dei professionisti: **Make** e **Git**.

## Perché automatizzare?

Salvatore spiega che dover riscrivere ogni volta `cc toyfort.c -Wall -O2 -o toyfort` è inefficiente. Inoltre, durante lo sviluppo, è facile fare modifiche che "rompono" il codice e voler tornare indietro.

### Il Makefile: La Ricetta della Compilazione

Un `Makefile` è un file di testo che contiene le "istruzioni" per compilare il programma. È basato su **Target** e **Prerequisiti**.

<CodeEditor 
  initialCode={`# Il target di default (all) dipende da toyfort
all: toyfort

# toyfort dipende dal sorgente .c
toyfort: toyfort.c
	cc toyfort.c -Wall -O2 -o toyfort

# Target per pulire i file generati
clean:
	rm -f toyfort`}
  language="makefile"
/>

<Callout type="warning" title="L'importanza del Tab">
Nei Makefile, le linee dei comandi **devono** iniziare con un carattere di Tabulazione reale, non con spazi! È uno dei pochi casi in cui il C (o meglio, gli strumenti intorno ad esso) è pignolo sugli spazi bianchi.
</Callout>

Salvatore racconta come, durante la registrazione, avesse cancellato per errore una parte di codice necessaria. Grazie a un backup dei transcript (o nel nostro caso, grazie a Git), è possibile recuperare versioni precedenti in pochi secondi.
</Callout>

## Evoluzione di ToyFort: `list_pop_type`

Mentre miglioriamo il workflow, miglioriamo anche l'interprete. Introduciamo una versione di `pop` che controlla il tipo dell'oggetto.

<CodeEditor 
  initialCode={`TFObject *list_pop_type(TFContext *ctx, int type) {
    if (ctx->stack_len == 0) return NULL;
    TFObject *o = ctx->stack[ctx->stack_len-1];
    
    // Se il tipo non coincide (e non abbiamo chiesto 'qualsiasi'), errore
    if (type != TF_ANY && o->type != type) return NULL;
    
    ctx->stack_len--;
    return o;
}`}
  language="c"
/>
