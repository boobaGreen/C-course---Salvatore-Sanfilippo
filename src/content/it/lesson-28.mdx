## Anatomia di una Funzione Variadica

Per dichiarare una funzione variadica, serve almeno un argomento "fisso" seguito dai tre puntini `...`.

```c
#include <stdarg.h>

void log_error(const char *fmt, ...) {
    va_list ap;
    va_start(ap, fmt); // Inizia dopo 'fmt'
    
    // ... estrazione argomenti ...
    
    va_end(ap); // Pulizia obbligatoria
}
```

<Callout type="warning" title="L'importanza di va_end">
Non dimenticare mai `va_end(ap)`. Anche se su molte architetture non fa nulla, su alcune piattaforme è fondamentale per evitare corruzione dello stack o memory leak.
</Callout>

---

## Il Pattern della Delegazione: `vprintf`

C'è un limite alle funzioni variadiche: non puoi passare i tre puntarelli (`...`) da una funzione all'altra come faresti con una variabile normale. Se vuoi scrivere un wrapper per `printf`, devi usare la sua versione "sorella" che accetta una `va_list`: la funzione `vprintf`.

## Esempio Pratico: Testo Centrato

Salvatore mostra come usare queste funzioni per manipolare l'output in modo creativo, ad esempio centrando il testo nel terminale.

<CodeEditor 
  initialCode={`void print_centered(int width, const char *fmt, ...) {
    char buffer[1024];
    va_list ap;
    
    va_start(ap, fmt);
    vsnprintf(buffer, sizeof(buffer), fmt, ap);
    va_end(ap);
    
    int len = strlen(buffer);
    int padding = (width - len) / 2;
    
    for (int i = 0; i < padding; i++) putchar(' ');
    printf("%s\\n", buffer);
}`}
  language="c"
/>

<Callout type="warning" title="Sicurezza dei Tipi">
Attenzione: `va_arg` non ha idea di cosa ci sia davvero nello stack. Se gli dici di estrarre un `int` ma l'utente ha passato una stringa, il programma non darà errore di compilazione, ma andrà in **Segmentation Fault** o corromperà la memoria a runtime!
</Callout>

## Estrarre gli Argomenti con `va_arg`

In C, la funzione non sa "quanti" argomenti le sono stati passati. Dobbiamo dirglielo noi tramite un formato (come nella printf) o un terminatore.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdarg.h>

void print_numbers(int count, ...) {
    va_list ap;
    va_start(ap, count);
    
    for (int i = 0; i < count; i++) {
        int num = va_arg(ap, int); // Specifica il TIPO!
        printf("Arg %d: %d\\n", i, num);
    }
    
    va_end(ap);
}

int main() {
    print_numbers(3, 10, 20, 30);
    return 0;
}`}
  language="c"
/>

<Callout type="hacker" title="Zero Type Safety">
Attenzione: `va_arg` non controlla se il tipo è giusto! Se passi un `double` ma chiedi un `int`, otterrai valori spazzatura o un crash. È responsabilità del programmatore garantire la coerenza.

<Callout type="danger" title="L'Attacco Format String">
  Mai passare una stringa ricevuta dall'utente direttamente come primo argomento di una funzione variadica (es. `printf(user_input)`). Un hacker potrebbe inserire simboli come `%n` per scrivere nella memoria del tuo programma e prenderne il controllo. Usa sempre `printf("%s", user_input)`.
</Callout>
</Callout>
