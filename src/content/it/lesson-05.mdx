## La verità dietro i caratteri

Quando scrivi `char c = 'A';`, il computer non salva la lettera "A", ma il numero **65**. Questo è il cuore del sistema ASCII.

<ASCIIConverter />

<Callout type="warning" title="Apici (') vs Virgolette ('')">
Questa è una delle differenze più "bastarde" per chi viene da Python o JS:
- `'A'` (apici singoli) è un **numero intero** (65). Occupa 1 byte.
- `"A"` (virgolette) è una **stringa**. È un array che contiene 'A' e un carattere invisibile `\0` (zero) per segnare la fine. Occupa 2 byte.
Scambiarli manderà in crisi il compilatore!

<Callout type="pro" title="Il Segreto del Byte Zero (\0)">
  In C, le stringhe non sanno quanto sono lunghe. Non c'è un campo "length"
  nascosto come in Java o Python. L'unico modo per sapere dove finisce una
  stringa è cercare il byte con valore **0** (binario `00000000`). **La Prova
  del Nove**: La stringa `"Ciao"` sembra lunga 4 caratteri, ma in RAM occupa **5
  byte**: `['C', 'i', 'a', 'o', '\0']` Dimenticare questo spazio extra per lo
  zero è la causa del 90% dei crash nei programmi C dei principianti!
</Callout>

<Callout type="info" title="Performance vs Sicurezza">
  Perché il C usa lo zero invece di salvare la lunghezza? Perché negli anni '70 risparmiare 1 o 2 byte per la lunghezza era vitale. Lo svantaggio? Per sapere quanto è lunga una stringa, le funzioni come `strlen()` devono scansionare ogni singolo carattere finché non trovano lo zero. Se lo zero manca, il programma continuerà a leggere la tua RAM all'infinito (o finché il sistema non lo blocca).
</Callout>
</Callout>

<Callout type="pro" title="Archeologia: Dal Telegrafo all'ASCII">
  L'ASCII (1963) è nato per standardizzare le macchine da scrivere telescopiche. Prima ogni produttore aveva il suo codice (come il codice Morse). L'ASCII ha fissato 128 simboli usando 7 bit. L'8° bit veniva spesso usato per il "controllo parità" (per vedere se c'erano stati errori di trasmissione sui vecchi cavi telefonici).
</Callout>
</Callout>

## Promozione dei Tipi

Sapevi che se passi un `char` a una funzione con argomenti variabili (come `printf`), il C lo promuove automaticamente a `int`? Ecco perché `%d` funziona anche per stampare il codice numerico di un carattere!

<CodeEditor
  initialCode={`#include <stdio.h>\n\nint main() {\n    char lettera = 'A'; // Un singolo intero\n    printf("Carattere: %c\\n", lettera);\n    printf("Codice ASCII: %d\\n", lettera);\n    \n    // Variadic promotion: char viene passato come int\n    printf("Casting manuale: %d\\n", (int)lettera);\n    return 0;\n}`}
