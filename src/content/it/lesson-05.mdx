## La verità dietro i caratteri

Quando scrivi `char c = 'A';`, il computer non salva la lettera "A", ma il numero **65**. Questo è il cuore del sistema ASCII.

<ASCIIConverter />

<Callout type="warning" title="Apici (') vs Virgolette ('')">
Questa è una delle differenze più "bastarde" per chi viene da Python o JS:
- `'A'` (apici singoli) è un **numero intero** (65). Occupa 1 byte.
- `"A"` (virgolette) è una **stringa**. È un array che contiene 'A' e un carattere invisibile `\0` (zero) per segnare la fine. Occupa 2 byte.
Scambiarli manderà in crisi il compilatore!

<Callout type="info" title="Il Terminatore NULL (\0)">
  In C, le stringhe non sanno quanto sono lunghe. L'unico modo per sapere dove finisce una stringa è cercare il byte con valore **0**. È una convenzione che ha reso il C velocissimo ma anche vulnerabile (se dimentichi lo zero, la `printf` continuerà a leggere la RAM finché non trova uno zero a caso o il programma crasha).
</Callout>

## Promozione dei Tipi

Sapevi che se passi un `char` a una funzione con argomenti variabili (come `printf`), il C lo promuove automaticamente a `int`? Ecco perché `%d` funziona anche per stampare il codice numerico di un carattere!

<CodeEditor 
  initialCode={`#include <stdio.h>\n\nint main() {\n    char lettera = 'A'; // Un singolo intero\n    printf("Carattere: %c\\n", lettera);\n    printf("Codice ASCII: %d\\n", lettera);\n    \n    // Variadic promotion: char viene passato come int\n    printf("Casting manuale: %d\\n", (int)lettera);\n    return 0;\n}`}

<KeyConcepts title="Hacker Key Concepts">
  - **I caratteri sono numeri**: Ogni carattere (`char`) è in realtà un intero a 8 bit.
  - **Tabella ASCII**: La mappa universale che assegna un numero univoco ad ogni simbolo testuale.
  - **Apici vs Virgolette**: `'A'` è un carattere (un numero), `"A"` è una stringa (un contenitore più complesso).
