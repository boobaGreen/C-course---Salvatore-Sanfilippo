## Lo Stack e la Logica di ToyFort

In ToyFort, se vuoi sommare 5 e 5, scrivi `5 5 +`. 
1. `5` viene messo nello stack.
2. Un altro `5` viene messo nello stack.
3. `+` preleva i due 5, li somma e mette `10` nello stack.

## Progettare gli Oggetti (TFObject)

Per rappresentare i dati di ToyFort (interi, stringhe, simboli, liste), usiamo le **Unioni** e il **Reference Counting** che abbiamo imparato nelle lezioni precedenti.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

typedef enum { TF_INT, TF_STR, TF_SYM, TF_LIST } TFType;

// L'oggetto principale di ToyFort
typedef struct TFObject {
    TFType type;
    int ref_count;
    union {
        long long i;
        struct { char *ptr; size_t len; } s; // Stringhe e Simboli
        struct { struct TFObject **ele; size_t len; } l; // Liste (Programmi)
    };
} TFObject;

// Stato del Parser durante la lettura del codice
typedef struct TFParser {
    char *prg;              // Il testo intero del programma
    char *next_token;       // Il punto dove siamo arrivati a leggere
} TFParser;

<Callout type="hacker" title="Perché i due asterischi?">
`TFObject **ele` è un puntatore a un array di puntatori. In C, per avere una lista che contiene oggetti di diverso tipo (eterogenea), non salviamo gli oggetti direttamente, ma i loro indirizzi. Rimuovendo un asterisco (`ele[0]`), otteniamo il puntatore al singolo oggetto.
</Callout>

// Contesto di Esecuzione (permette più interpreti isolati)
typedef struct TFContext {
    TFObject *stack;        // La nostra lista globale (usata come stack)
} TFContext;

// Funzione helper per allocazione sicura (xmalloc)
void* xmalloc(size_t size) {
    void *ptr = malloc(size);
    if (!ptr) { 
        fprintf(stderr, "Fatal: Out of memory\\n"); 
        exit(1); 
    }
    return ptr;
}

int main(int argc, char **argv) {
    if (argc < 2) {
        fprintf(stderr, "Usage: %s <program_file>\\n", argv[0]);
        return 1;
    }
    printf("Interprete ToyFort inizializzato per %s\\n", argv[1]);
    return 0;
}`} 
  language="c" 
/>

<Callout type="tip" title="xmalloc & Panic">
Salvatore introduce `xmalloc`: un wrapper intorno a `malloc` che interrompe il programma se la memoria finisce. È una pratica comune nei sistemi dove non è possibile gestire l'errore di memoria in modo sensato.
</Callout>

## Controllo del Flusso: IF-ELSE Stack-based

Salvatore spiega come implementare una condizionale usando le liste come "sottoprogrammi". In ToyFort, l'istruzione `IF` (o meglio `IF-ELSE`) preleva tre elementi dallo stack:
1. Una lista che rappresenta il ramo **falso**.
2. Una lista che rappresenta il ramo **vero**.
3. Un booleano (**condizione**).

<Callout type="info" title="Curiosità: argv[0] e Link Simbolici">
Durante la lezione, Salvatore fa un excursus su come i programmi (come `ls` o `vim`) possano cambiare comportamento in base al nome con cui vengono richiamati (usando `argv[0]`). Questo è possibile grazie ai link simbolici in Unix ed è una tecnica potente per il bootstrapping di sistemi.

### Memoria e Ricorsività
Nota come `TFObject` possa contenere una lista di altri `TFObject`. Questa struttura ricorsiva è fondamentale per rappresentare i programmi stessi: un programma ToyFort è una lista di "parole", che a loro volta possono essere liste (come i rami di un IF).
</Callout>
