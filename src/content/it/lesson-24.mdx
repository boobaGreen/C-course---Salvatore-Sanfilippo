## Il Cuore dell'Interprete: `eval()`

La funzione `eval` è responsabile di decidere cosa fare con un oggetto ToyFort. Se è un valore (intero, stringa), lo mette nello stack. Se è un simbolo, cerca la funzione nel dizionario e la esegue.

## Implementazione delle Funzioni (Built-ins)

Ogni funzione in ToyFort deve avere una corrispondenza in C. Queste funzioni interagiscono direttamente con il `TFContext` (lo stack).

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <string.h>

// Esempio di implementazione della funzione '+' (plus)
void tf_plus(TFContext *ctx) {
    // 1. Preleva gli ultimi due elementi (POP)
    TFObject *b = tf_stack_pop(ctx);
    TFObject *a = tf_stack_pop(ctx);

    if (a->type == TF_INT && b->type == TF_INT) {
        // 2. Calcola il risultato
        long long res = a->i + b->i;
        // 3. Crea un nuovo oggetto e mettilo nello stack (PUSH)
        tf_stack_push(ctx, create_int_object(res));
    } else {
        fprintf(stderr, "Runtime Error: + expects two integers\\n");
    }

    // 4. Gestione memoria (Reference Counting)
    tf_object_release(a);
    tf_object_release(b);
}
`} 
  language="c" 
/>

<Callout type="info" title="Complessità Ammortizzata">
Quando usiamo `realloc` per far crescere la lista (come visto in `list_push`), Salvatore spiega che allochiamo per potenze di due o comunque sovrallochiamo. Questo rende l'operazione di aggiunta **O(1) ammortizzato**, ovvero efficiente quanto una linked list ma con i vantaggi di velocità di un array.
</Callout>

<Callout type="warning" title="Memory Safety">
Ricorda: ogni volta che fai `pop` dallo stack, stai acquisendo la proprietà di quel puntatore. È fondamentale chiamare `release` (o `free`) alla fine della funzione per evitare memory leak!
</Callout>

<Callout type="danger" title="Sicurezza dell'Interprete">
  In un interprete reale, non puoi fidarti del codice dell'utente. Se il programma prova a fare `+` con un solo elemento nello stack, il tuo interprete crasherà cercando di accedere a un puntatore nullo. Un interprete "robusto" controlla sempre la profondità dello stack prima di ogni operazione.
</Callout>

## Parsing delle Liste e Programmi

In ToyFort, un programma è esso stesso una **lista** di oggetti. Questo permette di implementare condizionali e cicli passano intere liste come parametri ad altre funzioni.

### Il Parser a Mano
Salvatore implementa un parser "carattere per carattere". Funzioni come `skipSpaces` e `parsNumber` avanzano il puntatore all'interno della stringa sorgente. Questo approccio è lo stesso usato dai compilatori professionali (come GCC o Clang) per avere il massimo controllo sugli errori e sulla velocità.
