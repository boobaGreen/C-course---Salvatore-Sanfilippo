## Lo "Shock" Algorithmico

Fino ad ora abbiamo usato gli array. Gli array sono blocchi di memoria piatti. Ma cosa succede se non sappiamo quante righe ha un file? Invece di allocare un array gigantesco "a caso", usiamo una **Lista Linkata**.

Una lista linkata è come una serie di persone: ognuna tiene la mano di quella successiva. Se vuoi invertire l'ordine delle righe (come fa il comando `tac`), basta aggiungere ogni nuova riga "davanti" alla testa della lista.

### La Struttura de Nodo

```c
struct Line {
    char *text;          // Il contenuto della riga
    struct Line *next;   // Puntatore alla riga successiva (o NULL)
};
```

<Callout type="pro" title="Salto Mentale">
Nota che `struct Line` contiene un puntatore a `struct Line`. Questo è possibile perché un puntatore è solo un indirizzo di memoria; il compilatore non ha bisogno di sapere quanto è grande tutta la struct per definire il puntatore.
</Callout>

<Callout type="info" title="Inserimento in Testa">
Perché la lista inverte il file? Perché facciamo `new_node->next = head; head = new_node;`. Immagina di impilare dei piatti: l'ultimo che posi è il primo che prendi. Questa struttura si chiama **Stack** (Pila).
</Callout>

## Implementazione: Il comando TAC

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>
#include <string.h>

struct Line {
    char *text;
    struct Line *next;
};

int main(int argc, char **argv) {
    if (argc < 2) return 1;
    
    FILE *fp = fopen(argv[1], "r");
    if (!fp) return 1;

    char buf[1024];
    struct Line *head = NULL;

    // Leggiamo tutto il file e creiamo la lista "al contrario"
    while (fgets(buf, sizeof(buf), fp)) {
        struct Line *new_node = malloc(sizeof(struct Line));
        new_node->text = strdup(buf); // Alloca e copia la stringa
        new_node->next = head;        // Il vecchio 'head' diventa il 'next'
        head = new_node;              // Questo diventa il nuovo 'head'
    }

    // Stampiamo la lista (che ora è in ordine inverso!)
    struct Line *curr = head;
    while (curr) {
        printf("%s", curr->text);
        curr = curr->next;
    }

    // SFIDA: Aggiungi un ciclo finale per fare 'free()' di ogni nodo e della sua stringa!

    fclose(fp);
    return 0;
}`} 
  language="c" 
/>

<Callout type="info">
`strdup` è una funzione comodissima che fa `malloc` + `strcpy` in un colpo solo. Ricordati però che dovrai fare `free` sia di `text` che del nodo stesso!

<Callout type="danger" title="Puntatori Ciondolanti (Dangling)">
  Quando liberi un nodo con `free(curr)`, il puntatore `curr` contiene ancora l'indirizzo della memoria (ormai rubata dal SO). Usarlo dopo la `free` è catastrofico. Buona prassi: dopo ogni `free(p)`, assegna `p = NULL;`.
</Callout>
</Callout>
