## Mappare i Simboli: La Function Table

Ogni comando in ToyFort è una "entry" in una tabella. Quando l'interprete incontra un simbolo, lo cerca in questa lista e, se lo trova, chiama la funzione associata.

<Infographic 
  title="Lookup Flow"
  description="Come un simbolo diventa un'azione."
  items={[
    { label: "1. Match", value: "Cerca 'SUM' nella tabella", color: "blue" },
<Callout type="pro" title="Potenza delle SIMD">
  `memcmp` non confronta un byte alla volta. Le CPU moderne hanno istruzioni **SIMD** (come AVX o NEON) che possono confrontare 16, 32 o persino 64 byte in un solo colpo d'occhio. Usare le funzioni standard significa sfruttare gratuitamente tutta la potenza dell'hardware moderno.
</Callout>

<Callout type="info" title="String Interning">
  In molti interpreti, invece di comparare le stringhe ogni volta, si usa il "String Interning": si salva ogni stringa unica in una tabella globale e si usa solo il suo indirizzo. Se due indirizzi sono uguali, le stringhe sono uguali. Questo trasforma un confronto $O(N)$ in un istantaneo $O(1)$.
</Callout>

```c
int compare_string_objects(TFObject *a, TFObject *b) {
    size_t min_len = (a->s.len < b->s.len) ? a->s.len : b->s.len;
    int res = memcmp(a->s.ptr, b->s.ptr, min_len);
    if (res == 0) {
        if (a->s.len == b->s.len) return 0;
        return (a->s.len > b->s.len) ? 1 : -1;
    }
    return (res > 0) ? 1 : -1;
}
```
