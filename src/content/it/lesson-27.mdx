## Mappare i Simboli: La Function Table

Ogni comando in ToyFort è una "entry" in una tabella. Quando l'interprete incontra un simbolo, lo cerca in questa lista e, se lo trova, chiama la funzione associata.

<Callout type="pro" title="Potenza delle SIMD">
  `memcmp` non confronta un byte alla volta. Le CPU moderne hanno istruzioni
  **SIMD** (come AVX o NEON) che possono caricare registri da 256/512 bit e
  confrontare 32 o 64 caratteri in un solo colpo di clock. Usare le funzioni
  standard significa sfruttare gratuitamente tutta la potenza dell'hardware
  moderno.
</Callout>

## String Interning: Il Trucco della Memoria

In molti interpreti (Python, JavaScript, Lua), invece di comparare il contenuto delle stringhe ogni volta, si usa il "String Interning":

1. Si crea una tabella globale di tutte le stringhe uniche.
2. Se due stringhe hanno lo stesso indirizzo di memoria, sono garantite essere uguali.

### Implementazione: Comparazione Robusta

Poiché i nostri oggetti `TF_STR` salvano la propria lunghezza, la comparazione deve gestire anche byte nulli intermedi.

```c
int compare_string_objects(TFObject *a, TFObject *b) {
    if (a->s.len != b->s.len) return (a->s.len > b->s.len) ? 1 : -1;
    // Grazie alla lunghezza certa, usiamo memcmp (SIMD optimized)
    return memcmp(a->s.ptr, b->s.ptr, a->s.len);
}
```
