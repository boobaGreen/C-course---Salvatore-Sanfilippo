## Lo Scope delle Variabili: Dove "vive" il tuo codice?

In C, non tutte le variabili sono uguali. La loro "visibilità" e la loro "durata" dipendono da dove le dichiariamo.

### 1. Variabili Locali
Sono definite all'interno di una funzione. Nascono quando la funzione viene chiamata e **muoiono** quando la funzione termina. Se chiami la funzione più volte, ogni volta viene creata una nuova copia da zero.

### 2. Variabili Globali
Sono definite all'esterno di tutte le funzioni. Esistono per tutta la durata del programma e sono accessibili da qualsiasi punto del codice.

### 3. Variabili Statiche
Usando la keyword `static` dentro una funzione, creiamo una variabile che è "globale" nella durata (non perde il valore tra una chiamata e l'altra) ma "locale" nella visibilità (solo quella funzione può vederla).

## Passaggio per Valore: La Copia Silenziosa

Un concetto fondamentale del C è che gli argomenti delle funzioni vengono passati **per valore**. Questo significa che la funzione riceve una **copia** del dato, non l'originale.

```c
int incr(int x) {
    x = x + 1;
    return x;
}

int main() {
    int a = 10;
    incr(a); // Passiamo una COPIA di 'a'
    printf("%d", a); // Stampa ancora 10!
}
```

## Tipi Numerici e Promozioni

Il C offre una grande varietà di tipi per risparmiare memoria (fondamentale negli anni '60!).

### La "Gerarchia" dei Tipi
- `char`: 1 byte (8 bit)
- `short`: Tipicamente 2 byte
- `int`: Tipicamente 4 byte (ma dipende dal sistema)
- `float`: Numeri decimali a singola precisione
- `double`: Numeri decimali a doppia precisione (64 bit)

### Promozioni Automatiche
Quando fai operazioni tra tipi diversi, il C applica delle regole di "promozione". Questo accade perché il C cerca di rendere il calcolo più fluido (e perché negli anni '60 serviva a semplificare il lavoro del processore).

Ad esempio, in funzioni come `printf` (funzioni variadiche):
- Gli `short` e i `char` vengono promossi a `int`.
- I `float` vengono promossi a `double`.

## Esadecimale e Formattazione Avanzata
Sapevi che puoi stampare un numero in base 16 (esadecimale) semplicemente cambiando una lettera?
- `%d`: Decimale (base 10)
- `%x`: Esadecimale (base 16)

```c
int a = 10;
printf("Decimale: %d\n", a);   // Stampa 10
printf("Esadecimale: %x\n", a); // Stampa a
```

<Callout type="danger" title="Mischiare i Tipi è Disastroso">
Il C è un linguaggio "pericoloso": se dici alla `printf` di stampare un float (`%f`) ma gli passi un `int`, il programma leggerà la memoria in modo errato, stampando valori senza senso o crashando. Il compilatore ti darà un **warning**, ma ti lascerà compilare!
</Callout>

## Attenzione all'Overflow!

Cosa succede se aggiungi 1 al massimo valore possibile per un tipo?
- **Unsigned (senza segno)**: Il comportamento è garantito. Il valore "riparte" da zero (wrapping).
- **Signed (con segno)**: È un **Undefined Behavior**. Il compilatore potrebbe fare letteralmente qualsiasi cosa. Non fare mai affidamento sull'overflow dei numeri con segno!

<KeyConcepts title="In sintesi">
  - Lo **Scope** determina dove e per quanto tempo una variabile esiste.
  - Il C usa il **Passaggio per Valore** come standard.
  - I tipi numerici hanno limiti precisi e regole di **Promozione**.
  - Evita sempre l'overflow dei tipi con segno!
</KeyConcepts>
