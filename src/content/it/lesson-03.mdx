## Passaggio per Valore: La Copia Silenziosa

Un concetto fondamentale del C è che gli argomenti delle funzioni vengono passati **per valore**. Questo significa che la funzione riceve una **copia** del dato, non l'originale.

```c
int incr(int x) {
    x = x + 1;
    return x;
}

<Callout type="warning" title="= vs ==">
  Fai attenzione: `=` è l'operatore di **assegnamento** (sposta un valore), mentre `==` è l'operatore di **confronto**. Scrivere `if (x = 5)` invece di `if (x == 5)` è uno degli errori più comuni e bastardi del C, perché assegnerà 5 a x e la condizione risulterà sempre vera!
</Callout>

int main() {
    int a = 10;
    incr(a); // Passiamo una COPIA di 'a'
    printf("%d", a); // Stampa ancora 10!
}
```

<Infographic
  src="/images/pass_by_value.png"

  caption="Il C copia il valore nello stack della funzione chiamata. L'originale rimane invariato."
/>

## Tipi Numerici e Promozioni

Il C offre una grande varietà di tipi per risparmiare memoria (fondamentale negli anni '60!).

### La "Gerarchia" dei Tipi

- `char`: 1 byte (8 bit)
- `short`: Tipicamente 2 byte
- `int`: Tipicamente 4 byte (ma dipende dal sistema)
- `float`: Numeri decimali a singola precisione
- `double`: Numeri decimali a doppia precisione (64 bit)

<TypeMatcher />

<Infographic
  src="/images/integer_ranges.png"
  alt="Range dei tipi interi"
  caption="Cheat sheet dei tipi comuni e dei loro limiti di memoria."
/>

### Promozioni Automatiche

Quando fai operazioni tra tipi diversi, il C applica delle regole di "promozione". Questo accade perché il C cerca di rendere il calcolo più fluido (e perché negli anni '60 serviva a semplificare il lavoro del processore).

Ad esempio, in funzioni come `printf` (funzioni variadiche):

- Gli `short` e i `char` vengono promossi a `int`.
- I `float` vengono promossi a `double`.

Questo significa che, tecnicamente, la `printf` non "sa" stampare un float: riceve sempre un double e lo tratta come tale.

<Callout type="info" title="Side Effects (Effetti Collaterali)">
  Una funzione `void` che non prende nulla e non restituisce nulla (come `void
  hello(void)`) ha valore solo per i suoi **side effects**: stampare a video,
  scrivere su disco o accendere un LED. In programmazione funzionale pura sono
  "peccati", in C sono il pane quotidiano.
</Callout>

## Esadecimale e Formattazione Avanzata

Sapevi che puoi stampare un numero in base 16 (esadecimale) semplicemente cambiando una lettera?

- `%d`: Decimale (base 10)
- `%x`: Esadecimale (base 16)

```c
int a = 10;
printf("Decimale: %d\n", a);   // Stampa 10
printf("Esadecimale: %x\n", a); // Stampa a
```

<Callout type="danger" title="Mischiare i Tipi è Disastroso">
  Il C è un linguaggio "pericoloso": se dici alla `printf` di stampare un float
  (`%f`) ma gli passi un `int`, il programma leggerà la memoria in modo errato,
  stampando valori senza senso o crashando. Il compilatore ti darà un
  **warning**, ma ti lascerà compilare!
</Callout>

<Callout type="warning" title="La Trappola della Promozione">
  Sapevi che se confronti un `int` negativo con un `unsigned int`, il risultato potrebbe essere... falso?
  `if (-1 > 1u)` risulta **VERO** in C. Perché? Perché `-1` viene "promosso" al tipo più grande (`unsigned`), diventando un numero enorme (es. 4 miliardi). Mai mischiare signed e unsigned nei confronti!
</Callout>

## Attenzione all'Overflow!

Cosa succede se aggiungi 1 al massimo valore possibile per un tipo?

- **Unsigned (senza segno)**: Il comportamento è garantito. Il valore "riparte" da zero (wrapping).
- **Signed (con segno)**: È un **Undefined Behavior**. Il compilatore potrebbe fare letteralmente qualsiasi cosa. Non fare mai affidamento sull'overflow dei numeri con segno!
