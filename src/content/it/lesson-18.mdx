## Usare Typedef: Più di un Semplice Alias

`typedef` non crea un nuovo tipo, ma un "soprannome" per un tipo esistente. Salvatore lo consiglia caldamente per evitare di spargere la parola chiave `struct` ovunque.

```c
typedef struct {
    int x;
    int y;
} Point;

Point p; // Molto più pulito di 'struct Point p'
```

### Struct Anonime e Pulizia

Notate che nella `typedef` sopra non abbiamo dato un nome alla struct dopo la parola chiave `struct`. È una **Struct Anonima**: ci serve solo il nome dell'alias `Point` per usare l'oggetto nel programma.

## L'API dei File (stdio.h): Lo Stream di Byte

In C, un file è visto come uno **stream** (flusso) infinito di byte. La libreria standard ci fornisce un puntatore a una struttura speciale chiamata `FILE`.

<Callout type="pro" title="Tipi Opachi: Il segreto del Design Professionale">
  Il tipo `FILE` è un **Tipo Opaco**. Tu non sai cosa c'è dentro la struct
  `FILE` (e non ti è permesso vederlo!). Usi solo il puntatore e le funzioni
  della libreria. Questo permette agli sviluppatori del sistema operativo di
  cambiare l'implementazione interna dei file (magari per renderli 10x più
  veloci) senza che il tuo codice debba essere ricompilato. È l'essenza
  dell'**Incapsulamento**.
</Callout>

<Callout type="warning" title="Dimenticare fclose()">
  Ogni file aperto occupa un **File Descriptor** nel Kernel. Se non fai
  `fclose`, rischi di esaurire le risorse del sistema. Inoltre, i dati scritti
  con `fwrite` rimangono spesso in un "Buffer di scrittura" nella RAM: se il
  programma crasha prima della chiusura, quei dati andranno persi per sempre
  perché non sono mai arrivati fisicamente sul disco!
</Callout>

## Testo vs Binario: Perché il Binario Vince

Quando salviamo un punteggio di un gioco, possiamo scriverlo come testo (`"100"`) o come binario (`0x00000064`).

<Callout type="pro" title="Persistenza dell'Oggetto">
  Usare `fwrite(&player, sizeof(Player), 1, fp)` è il modo più veloce per
  salvare lo stato di un programma. Stai letteralmente "fotografando" la RAM e
  scrivendola su un file. Quando riapri il file con `fread`, l'oggetto rinasce
  esattamente com'era.
</Callout>

## Esercitazione

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdint.h>

typedef struct {
uint32_t id;
float score;
} Player;

int main() {
Player p1 = {1, 95.5f};

    // Scrittura Binaria
    FILE *fp = fopen("save.dat", "wb");
    if (fp) {
        fwrite(&p1, sizeof(Player), 1, fp);
        fclose(fp);
        printf("Salvataggio binario completato (8 byte).\\n");
    }

    // Lettura Binaria
    Player p2;
    fp = fopen("save.dat", "rb");
    if (fp) {
        fread(&p2, sizeof(Player), 1, fp);
        printf("Caricato Player ID: %d, Score: %.1f\\n", p2.id, p2.score);
        fclose(fp);
    }

    return 0;

}`}
