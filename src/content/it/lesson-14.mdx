## Perché le Struct? L'Esempio delle Frazioni

Immaginate di voler gestire migliaia di frazioni. Senza le struct, dovreste avere migliaia di variabili `num` e `den`. Un incubo. Salvatore ci mostra come una `struct` risolve il problema:

```c
struct Frazione {
    int num;  // Numeratore
    int den;  // Denominatore
};
```

### Anatomia di un Oggetto in Memoria

Una struct è un blocco di memoria contiguo. Se un `int` occupa 4 byte, la nostra `struct Frazione` occuperà esattamente **8 byte**.

## Il Mistero del Padding (Hacker Knowledge)

Cosa succede se aggiungiamo un `char` (1 byte) alla nostra struct? Ci aspetteremmo una dimensione di 9 byte (4 + 4 + 1). Ma il C ci sorprende: la dimensione sarà **12 byte**.

<Callout type="warning" title="Ottimizzazione">
Per risparmiare memoria, Salvatore consiglia di raggruppare i campi per dimensione: prima i più grandi (puntatori, long, int) e poi i più piccoli (char). Questo riduce il padding inserito dal compilatore!

<Callout type="hacker" title="Attribute Packed">
  In casi estremi (es. pacchetti di rete che devono essere identici al millimetro), puoi dire al GCC: `__attribute__((packed))`. Questo forza il compilatore a rimuovere **tutto** il padding. Attenzione: il programma diventerà leggermente più lento perché la CPU dovrà fare più fatica a leggere i dati disallineati.
</Callout>
</Callout>

<Callout type="info" title="L'origine della Freccia ->">
Perché non usare sempre il punto? Nei primi anni del C, il compilatore aveva bisogno di un indizio sintattico forte per capire se stavi accedendo a una locazione fissa (punto) o se doveva prima calcolare un indirizzo base da un registro (freccia). Oggi è una scelta di stile che rende il codice estremamente leggibile: vedi una freccia, sai che c'è un puntatore di mezzo.
</Callout>

## Operazioni: Punto vs Freccia

È la confusione classica di ogni studente C. Ecco la regola definitiva di Salvatore scavata nella filosofia del linguaggio:

- **Operatore Punto (`.`)**: Si usa quando hai la **variabile fisica** (nello stack o come copia).
- **Operatore Freccia (`->`)**: Si usa quando hai un **puntatore** (indirizzo memoria). È lo shorthand di `(*ptr).campo`.

<CodeEditor 
  initialCode={`#include <stdio.h>
#include <stdlib.h>

struct Frac {
    int num;
    int den;
};

// Funzione che accetta un PUNTATORE (Passaggio per Referenza)
void simplify(struct Frac *f) {
    if (f->num == f->den) {
        f->num = 1;
        f->den = 1;
    }
}

int main() {
    // 1. Variabile fisica
    struct Frac a;
    a.num = 10;
    a.den = 10;
    
    // 2. Usiamo il puntatore
    simplify(&a);
    
    printf("Frazione semplificata: %d/%d\\n", a.num, a.den);

    // SFIDA: Aggiungi qui una funzione 'multiply' che prende due struct Frac e ne ritorna una nuova!
    
    // 3. Allocazione dinamica nell'HEAP
    struct Frac *b = malloc(sizeof(struct Frac));
    b->num = 3;
    b->den = 4;
    
    printf("Frazione nell'heap: %d/%d\\n", b->num, b->den);
    
    free(b);
    return 0;
}`}

### Riassunto Finale
Le strutture sono il cuore pulsante del C professionale. Ci permettono di modellare la realtà (frazioni, utenti, pacchetti di rete) rispettando la ferrea logica della memoria fisica. Ricorda: l'ordine dei campi conta per il padding, e la freccia è la chiave per manipolare oggetti dinamici!
